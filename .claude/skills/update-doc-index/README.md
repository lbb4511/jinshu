# update-doc-index 技能使用指南

自动扫描文档目录结构，生成/更新文档总索引，保持文档目录的准确性。

## 用法

```bash
/update-doc-index
```

## 功能

### 1. 自动生成文档总索引
扫描`docs/`目录下的所有markdown文件，自动生成：
- 📁 目录结构总览（树形目录）
- 📊 文档统计（按分类统计完成率）
- 🔗 快速链接索引表（按目录分组）
- ✅ 自动识别文档状态（已完成/待写）

### 2. 检查链接有效性
验证所有文档间的相对链接是否有效，检测：
- 指向不存在文件的死链接
- 路径错误的链接
- 重复链接

### 3. 自动更新文档完成率
根据文件存在情况自动更新各文档目录的完成百分比。

## 示例

```bash
# 生成/更新文档索引
/update-doc-index

# 检查现有索引与实际文件的差异
/update-doc-index --check

# 预览结果，不实际修改
/update-doc-index --dry-run

# 只输出目录树结构
/update-doc-index --tree
```

## 文档状态识别规则

| 状态标识 | 判断条件 |
|---------|---------|
| ✅ 已完成 | 文件存在，且文件大小 > 1KB |
| 📝 草稿 | 文件存在，但文件大小 < 1KB |
| ⏳ 待写 | 文件不存在，仅在README中有条目 |

## 目录命名规范

为确保正确分类，目录命名遵循以下规范：

```
docs/
├── 01-立项与架构/       # 🔴 顶层设计（必须100%完成才开始编码）
├── 02-流程与规范/       # 🟡 开发流程与规范（必须100%完成才开始编码）
├── 04-模块设计/         # 🟢 业务模块设计
├── 05-功能设计/         # 🟢 单个功能点设计
├── 03-安全与合规/       # 🔐 安全相关（必须100%完成才开始编码）
├── 06-运维与部署/       # ☁️ 运维相关
├── 07-测试/             # 🧪 测试相关
└── 99-附录/             # 📎 参考资料、FAQ
```

## 文档编号规范

每个目录下的文件统一使用两位数字编号：
```
01.xxxx.md
02.xxxx.md
...
09.xxxx.md
10.xxxx.md
```

特殊文件：
- `README.md` - 各目录的索引文件，不需要编号
- `00.快速入门.md` - 该目录的入门指南，放在最前面

## 常见问题

### Q: 如何标记文档为"待写"？
在目录的README.md中列出计划中的文档，不带markdown链接即可：
```markdown
| 03.报表导出模块设计.md | ⏳ 待写 | 百万行导出、CSV ZIP压缩 |
```

### Q: 如何标记文档为"草稿"？
创建文件但只写标题和TODO，内容少于1KB会自动识别为草稿状态。

### Q: 新增目录后需要手动更新索引吗？
不需要，运行`/update-doc-index`会自动扫描所有子目录并更新总索引。

## 建议使用场景

1. **每次新增文档后** - 运行更新，保持索引同步
2. **每次PR前** - 运行检查，确保文档链接都正确
3. **每周一** - 定期运行，统计文档完成进度

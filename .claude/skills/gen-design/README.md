# gen-design 技能使用指南

根据项目标准模板自动生成模块/功能设计文档，确保所有设计文档结构统一，包含最佳实践。

## 用法

```bash
/gen-design "文档标题" [--type module|feature]
```

## 示例

```bash
# 生成模块设计文档
/gen-design "报表导出模块" --type module

# 生成功能设计文档
/gen-design "大任务自动分片功能" --type feature

# 同时生成SQL DDL模板
/gen-design "任务调度模块" --sql

# 同时生成API设计模板
/gen-design "批量导入功能" --api

# 仅显示模板
/gen-design --template
```

## 文档类型

### --type module (默认)
模块级设计，适用于完整的业务模块，包含：
- 完整的架构设计
- 完整的数据库表设计
- API接口设计
- 完整的测试方案

### --type feature
功能级设计，适用于单个功能点，包含：
- 功能流程设计
- 接口变更说明
- 兼容性分析

## 标准模板结构

生成的设计文档自动包含以下章节：

### 1. 文档信息表头
| 项 | 内容 |
|----|------|
| 功能名称 | |
| 文档版本 | v1.0 |
| 创建日期 | YYYY-MM-DD |
| 设计人 | |
| 评审状态 | 待评审 |

### 2. 背景与目标
- 问题背景
- 设计目标
- 非目标（明确不做什么）

### 3. 需求分析
- 功能需求清单
- 用户故事
- 边界场景

### 4. 架构/流程设计
- 整体流程图
- 模块划分
- 关键算法/策略

### 5. 数据库设计（自动包含多租户字段）
```sql
CREATE TABLE xxx (
    id          BIGSERIAL PRIMARY KEY,
    tenant_id   BIGINT NOT NULL,
    ...
    created_at  TIMESTAMP DEFAULT NOW(),
    updated_at  TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_xxx_tenant ON xxx(tenant_id);
```

### 6. API接口设计
自动生成标准的RESTful API模板，包含：
- 接口列表
- 请求/响应示例
- 错误码定义

### 7. 异常处理设计
标准异常处理模板

### 8. 性能考量
性能目标、优化点、压测方案

### 9. 安全考量
权限控制、数据隔离、审计日志

### 10. 测试方案
单元测试、集成测试、边界测试

### 11. 上线方案
灰度策略、回滚方案、迁移方案

## 自动生成的内容

### 多租户相关（自动插入）
- 所有业务表自动包含`tenant_id`字段
- 所有索引以`tenant_id`作为最左前缀
- `tenant_id`过滤规则说明

### 审计日志相关（自动插入）
- 操作记录表模板
- 防篡改触发器模板
- 三权分立设计要点

### 错误码设计模板
自动填充标准错误码定义：
```
SUCCESS = 0
PARAMETER_ERROR = 400
UNAUTHORIZED = 401
FORBIDDEN = 403
NOT_FOUND = 404
SYSTEM_ERROR = 500
```

## 生成文件位置

| 类型 | 存储位置 |
|------|---------|
| 模块设计 | `docs/04-模块设计/XX.模块名称.md` |
| 功能设计 | `docs/05-功能设计/XX.功能名称.md` |

## 使用流程

1. **生成设计文档**
   ```bash
   /gen-design "报表PDF渲染模块" --sql --api
   ```

2. **补充内容**
   - 填充具体的业务逻辑
   - 补充特殊的表设计
   - 补充API细节

3. **提交评审**
   - PR提交设计文档
   - 等待评审意见
   - 根据评审意见修改

4. **评审通过后开始编码**

## 设计文档完成标准（摘自规范）：
> 一个合格的开发者拿到设计文档就能开始编码，不需要再问"这个地方怎么做"这类问题。

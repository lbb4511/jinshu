# review-test - 测试用例与测试数据评审技能

自动评审测试用例的完整性、覆盖率和测试数据的合规性，确保测试质量。

## 评审范围

### ✅ 测试用例完整性检查
- 是否覆盖正常流程
- 是否覆盖异常流程
- 是否包含边界值测试
- 是否包含并发场景测试
- 是否包含性能测试方案

### ✅ 多租户隔离测试检查
- 是否包含跨租户数据泄露测试
- 是否验证租户数据隔离
- 是否测试 `@SkipTenantFilter` 的安全影响
- 是否包含管理员全局查询的权限验证

### ✅ 测试数据合规性检查
- 测试数据是否脱敏（不包含真实用户数据）
- 是否包含 PII（个人身份信息）检测
- 是否符合数据安全法要求
- 是否包含边界值测试数据（空值、极值、超长字符串）

### ✅ 性能测试数据检查
- 性能测试数据量是否符合真实场景
- 是否包含百万级大数据量测试
- 是否有租户配额限制的测试数据
- 是否包含并发用户数的合理设计

### ✅ 安全测试检查
- 是否包含 SQL 注入测试
- 是否包含权限绕过测试
- 是否包含敏感数据泄露测试
- 是否包含审计日志完整性验证

### ✅ 单元测试规范检查
- 是否符合 F.I.R.S.T 原则（Fast, Independent, Repeatable, Self-validating, Timely）
- 测试命名是否规范
- 是否包含断言
- Mock 使用是否合理

## 使用方法

```bash
# 评审单个测试文件
/review-test backend/src/test/java/com/jinshu/ReportImportTest.java

# 评审整个测试目录
/review-test backend/src/test/java/

# 评审测试数据文件
/review-test test-data/ --data

# 严格模式（不通过则返回错误）
/review-test backend/src/test/java/ --strict

# 仅输出评审摘要
/review-test backend/src/test/java/ --summary

# 生成评审报告文件
/review-test backend/src/test/java/ --report
```

## 评审等级

| 等级 | 颜色 | 说明 | 处理方式 |
|------|------|------|---------|
| P0 | 🔴 严重 | 缺少关键测试场景或测试数据合规性问题 | 必须修复，不允许合并代码 |
| P1 | 🟡 重要 | 测试覆盖率不足或边界情况遗漏 | 建议修复后再提交 PR |
| P2 | 🟢 建议 | 可以优化的测试点 | 可在后续迭代中完善 |

## 标准通过条件

测试评审通过必须满足：
1. ✅ 核心业务流程 100% 覆盖
2. ✅ 包含多租户隔离测试
3. ✅ 测试数据完全脱敏，无真实用户信息
4. ✅ 包含异常流程和边界值测试
5. ✅ 包含并发场景测试
6. ❌ 无 P0 级别问题
7. ❌ P1 级别问题不超过 3 个

## 测试数据敏感信息检测清单

自动检测以下敏感信息：
- 手机号：11 位数字，符合手机号段规则
- 身份证号：18 位身份证号格式
- 银行卡号：16-19 位数字
- 邮箱地址：含 @ 的邮箱格式
- 真实姓名：常见中文姓名 2-4 字
- 地址信息：省/市/区/街道 等地址关键字
- 密码/密钥：password, secret, key 等关键字段

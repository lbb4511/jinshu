## 1. 架构分层原则

| 分层原则 | 说明                                 |
| -------- | ------------------------------------ |
| 单一职责 | 每层只做一件事                       |
| 单向依赖 | 上层依赖下层，禁止反向依赖           |
| 可测试   | 每层都可以独立单元测试               |
| 可替换   | 层与层之间通过接口交互，便于替换实现 |

## 3. 各层职责说明

### 3.1 Controller 层（接入层）

**职责**：

1. 接收 HTTP 请求，参数校验（JSR-380）
2. 调用 Service 层方法
3. 统一包装 Result 返回
4. 权限注解校验（`@PreAuthorize`）

**编码规范**：

- ✅ 只能依赖 Service 层
- ✅ 不允许出现业务逻辑
- ✅ 方法参数必须加 `@Valid` 校验
- ❌ 禁止直接注入 `SqlSession` 或 `Mapper`
- ❌ 禁止手动 try-catch 异常

**示例**：

```java
@GetMapping("/{id}")
public Result<Report> getById(@PathVariable Long id) {
    Report report = reportService.getById(id);
    return Result.success(report);
}
```

---

### 3.2 Service 层（业务服务层）

**职责**：

1. 核心业务逻辑实现
2. 事务控制（`@Transactional`）
3. 调用 DAO 层操作数据库
4. 发布领域事件（异步解耦）

**编码规范**：

- ✅ 只能依赖 DAO 层和基础设施层
- ✅ 业务校验失败抛出 `BusinessException`
- ✅ 复杂业务逻辑拆分为 private 方法
- ❌ 禁止出现任何 HTTP/Servlet 相关对象
- ❌ 禁止直接操作 `ThreadLocal`（用 Context 工具类）

**示例**：

```java
@Transactional
public void delete(Long id) {
    Report report = this.getById(id);
    if (report.getStatus() == ReportStatus.PUBLISHED) {
        throw new BusinessException(ErrorCode.REPORT_ALREADY_PUBLISHED);
    }
    reportMapper.deleteById(id);
}
```

---

### 3.3 DAO 层（数据访问层）

**职责**：

1. 数据库单表 CRUD
2. 复杂 SQL 查询
3. 分页查询

**编码规范**：

- ✅ 只能操作数据库
- ✅ 方法名清晰表达查询意图
- ❌ 禁止出现业务逻辑
- ❌ 禁止事务注解

**示例**：

```java
@Select("SELECT * FROM report WHERE tenant_id = #{tenantId} AND created_at BETWEEN #{start} AND #{end}")
List<Report> selectByTimeRange(Long tenantId, LocalDateTime start, LocalDateTime end);
```

---

### 3.4 跨层调用约束

```
✅ 允许：Controller → Service → DAO
✅ 允许：Service → Service（同模块内）
❌ 禁止：Controller → DAO（跳过 Service）
❌ 禁止：DAO 调用 Service
❌ 禁止：跨模块 Service 直接调用（通过 API 接口）
```

## 4. 工程结构与目录映射

### 4.1 Gradle Multi-Project 总览

Java 后端采用 Gradle multi-project 构建（决策见 `03.技术选型决策记录.md` 决策 006），与 `02.概要设计.md` §4.1 的 K8s 服务一一对应：

```
backend/                            # Gradle root project
├── settings.gradle.kts             # 声明 5 个子模块
├── build.gradle.kts                # 公共依赖、Java 版本、Spring Boot BOM
├── gradle/
│   └── libs.versions.toml          # 版本号集中管理
├── common/                         # 公共模块，不可独立启动，被其它 4 个模块依赖
├── api/                            # → K8s service: backend
├── batch/                          # → K8s service: worker（Spring Batch ETL 导入）
├── worker/                         # → K8s service: worker（MQ 消费，导出/PDF 编排）
└── data-sync/                      # → K8s service: data-sync-service
```

依赖方向（单向）：

```
api ──┐
batch ┤
worker┼──→ common
data-sync ┘
```

任何子模块**禁止**横向依赖另一个非 common 子模块。跨模块协作走 HTTP REST 或 MQ。

### 4.2 各子模块内部分层

**common**（无主类，jar 制品）：

```
com.jinshu.common
├── entity/          # Tenant、User、Report 等领域实体
├── result/          # Result、PageResult
├── exception/       # BusinessException、ErrorCode、GlobalExceptionHandler
├── context/         # TenantContext、UserContext（ThreadLocal 封装）
├── utils/           # HashUtils 等纯工具
└── security/        # TenantInterceptor、SkipTenantFilter、AuditLogAspect 等可被多模块复用的切面
```

**api**（主类 `ApiApplication`，K8s service: backend）：

```
com.jinshu.api
├── ApiApplication.java
├── controller/      # 所有 HTTP 接入
├── service/         # 业务编排
├── dao/             # MyBatis Mapper
├── config/          # MyBatisConfig、SecurityConfig、Web 相关
└── resources/
    ├── application.yml
    └── mapper/      # *.xml
```

**batch**（主类 `BatchApplication`，K8s service: worker）：

```
com.jinshu.batch
├── BatchApplication.java
├── job/             # Spring Batch JobConfiguration
├── reader/ writer/ processor/
├── service/         # 导入业务编排（三级校验、错误日志写入）
├── dao/             # 仅本模块用的 Mapper
└── resources/application.yml
```

**worker**（主类 `WorkerApplication`，K8s service: worker）：

```
com.jinshu.worker
├── WorkerApplication.java
├── consumer/        # MQ 监听器（导出任务、PDF 编排任务）
├── service/         # 编排逻辑
├── pdf/             # 调 renderer 服务的客户端 + PDFBox 合并、Ghostscript CMYK 转换
├── export/          # EasyExcel/CSV 流式写入
└── resources/application.yml
```

**data-sync**（主类 `DataSyncApplication`，K8s service: data-sync-service）：

```
com.jinshu.datasync
├── DataSyncApplication.java
├── controller/      # REST API 给 batch/worker 调用
├── service/         # 跨库同步编排
├── connector/       # 各类数据源连接器抽象
├── config/          # 数据源配置（K8s Secret 注入）
└── resources/application.yml
```

### 4.3 跨子模块调用约束

§3 已声明的 `Controller → Service → DAO` 单向依赖，在每个非 common 子模块内**独立适用**。

跨子模块禁止 Java 直接调用，统一走：

- HTTP REST（如 `worker → data-sync`、`api → data-sync`）
- MQ（如 `api → worker`、`api → batch`）

## 5. 数据流

```
1. HTTP Request
   ↓
2. Spring Validation 校验参数
   ↓
3. Spring Security 权限校验
   ↓
4. TenantInterceptor 设置租户上下文
   ↓
5. Controller 接收参数，调用 Service
   ↓
6. Service 执行业务逻辑，事务控制
   ↓
7. DAO 执行 SQL（MyBatis 插件自动注入 tenant_id）
   ↓
8. Service 返回结果给 Controller
   ↓
9. Controller 包装 Result 返回
   ↓
10. GlobalExceptionHandler 捕获异常（如有）
```

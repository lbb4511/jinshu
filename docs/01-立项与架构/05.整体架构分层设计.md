## 1. 架构分层原则

| 分层原则 | 说明                                 |
| -------- | ------------------------------------ |
| 单一职责 | 每层只做一件事                       |
| 单向依赖 | 上层依赖下层，禁止反向依赖           |
| 可测试   | 每层都可以独立单元测试               |
| 可替换   | 层与层之间通过接口交互，便于替换实现 |

## 3. 各层职责说明

### 3.1 Controller 层（接入层）

**职责**：

1. 接收 HTTP 请求，参数校验（JSR-380）
2. 调用 Service 层方法
3. 统一包装 Result 返回
4. 权限注解校验（`@PreAuthorize`）

**编码规范**：

- ✅ 只能依赖 Service 层
- ✅ 不允许出现业务逻辑
- ✅ 方法参数必须加 `@Valid` 校验
- ❌ 禁止直接注入 `SqlSession` 或 `Mapper`
- ❌ 禁止手动 try-catch 异常

**示例**：

```java
@GetMapping("/{id}")
public Result<Report> getById(@PathVariable Long id) {
    Report report = reportService.getById(id);
    return Result.success(report);
}
```

---

### 3.2 Service 层（业务服务层）

**职责**：

1. 核心业务逻辑实现
2. 事务控制（`@Transactional`）
3. 调用 DAO 层操作数据库
4. 发布领域事件（异步解耦）

**编码规范**：

- ✅ 只能依赖 DAO 层和基础设施层
- ✅ 业务校验失败抛出 `BusinessException`
- ✅ 复杂业务逻辑拆分为 private 方法
- ❌ 禁止出现任何 HTTP/Servlet 相关对象
- ❌ 禁止直接操作 `ThreadLocal`（用 Context 工具类）

**示例**：

```java
@Transactional
public void delete(Long id) {
    Report report = this.getById(id);
    if (report.getStatus() == ReportStatus.PUBLISHED) {
        throw new BusinessException(ErrorCode.REPORT_ALREADY_PUBLISHED);
    }
    reportMapper.deleteById(id);
}
```

---

### 3.3 DAO 层（数据访问层）

**职责**：

1. 数据库单表 CRUD
2. 复杂 SQL 查询
3. 分页查询

**编码规范**：

- ✅ 只能操作数据库
- ✅ 方法名清晰表达查询意图
- ❌ 禁止出现业务逻辑
- ❌ 禁止事务注解

**示例**：

```java
@Select("SELECT * FROM report WHERE tenant_id = #{tenantId} AND created_at BETWEEN #{start} AND #{end}")
List<Report> selectByTimeRange(Long tenantId, LocalDateTime start, LocalDateTime end);
```

---

### 3.4 跨层调用约束

```
✅ 允许：Controller → Service → DAO
✅ 允许：Service → Service（同模块内）
❌ 禁止：Controller → DAO（跳过 Service）
❌ 禁止：DAO 调用 Service
❌ 禁止：跨模块 Service 直接调用（通过 API 接口）
```

## 5. 数据流

```
1. HTTP Request
   ↓
2. Spring Validation 校验参数
   ↓
3. Spring Security 权限校验
   ↓
4. TenantInterceptor 设置租户上下文
   ↓
5. Controller 接收参数，调用 Service
   ↓
6. Service 执行业务逻辑，事务控制
   ↓
7. DAO 执行 SQL（MyBatis 插件自动注入 tenant_id）
   ↓
8. Service 返回结果给 Controller
   ↓
9. Controller 包装 Result 返回
   ↓
10. GlobalExceptionHandler 捕获异常（如有）
```

## 1. 概述

### 1.1 目的

定义锦书企业级报表系统在任何环境（开发/测试/预发/生产）上线前必须完成的安全检查项。全部 A 类检查项**必须通过**后方可上线，任一未通过则上线阻塞。

### 1.2 使用说明

| 检查结果 | 含义                                                   |
| -------- | ------------------------------------------------------ |
| ✅ PASS  | 检查通过                                               |
| ❌ FAIL  | 检查未通过，**上线阻塞**（A 类）或需记录原因（B/C 类） |
| ⚪ N/A   | 不适用当前环境                                         |
| ⚠️ WARN  | 结果处于临界值，建议改进但不阻塞上线                   |

**检查频率**：

| 环境          | 频率                     |
| ------------- | ------------------------ |
| 生产环境      | 每次上线前               |
| 预发环境      | 每次上线前               |
| 开发/测试环境 | 初期上线前一次，后续按需 |

| #   | 检查项             | 检查方法                                            | 通过标准                              | 结果 |

| A2  | TLS 证书有效性     | 浏览器访问 + `openssl s_client -connect <host>:443` | 证书未过期、域名匹配、TLS ≥ 1.2       |      |

## 4. 常见问题与解决方案

### 问题 1：TLS 证书过期

**现象**：浏览器提示"您的连接不是私密连接"

**解决**：

```bash
## 手动续期（cert-manager 自动续期失败时）
kubectl delete certificate -n jinshu jinshu-tls
kubectl apply -f k8s/certificate.yaml
```

### 问题 2：审计日志触发器未生效

**现象**：`UPDATE audit_log` 执行成功

**解决**：

- 检查 DDL 脚本中的触发器是否正确执行
- 确认数据库用户对 `audit_log` 表无 UPDATE/DELETE 权限
- 验证 `REVOKE UPDATE, DELETE ON audit_log FROM app_user;` 已执行

### 问题 3：依赖漏洞扫描发现高危漏洞

**现象**：Trivy 报告存在 HIGH 级别漏洞

**解决**：

1. 评估漏洞是否在当前部署环境可被利用
2. 如可利用 → 升级到修复版本，重新构建镜像
3. 如不可利用 → 记录风险评估理由，标记为 WARN 通过

### 问题 4：前端 Token 存储在 localStorage

**现象**：检查发现 JWT 存储在 `localStorage`

**解决**：

1. 改用 `httpOnly Secure SameSite=Strict` Cookie
2. 如必须使用 Web Storage，评估 XSS 风险并配置严格 CSP

### 问题 5：密码策略未在前端校验

**现象**：可通过前端发送弱密码

**解决**：

1. 前端添加密码强度实时校验（≥8 位，含大小写+数字+特殊字符至少三种）
2. 后端也必须独立校验（纵深防御）

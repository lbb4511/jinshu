## 1. 背景与目标

### 1.1 问题背景

API 是系统对外暴露的攻击面。当前系统缺少统一的 API 安全防护策略：

| 问题             | 风险                               |
| ---------------- | ---------------------------------- |
| 无限流机制       | 单个用户/租户可能拖垮系统          |
| 无请求体大小限制 | 恶意大文件上传可能导致 OOM         |
| 安全头未配置     | 浏览器层面的 XSS/Clickjacking 风险 |
| CORS 策略未严格  | 跨域攻击面                         |
| 文件上传未校验   | 恶意文件上传（WebShell/病毒）风险  |

### 1.2 设计目标

| 指标       | 要求                                     |
| ---------- | ---------------------------------------- |
| 整体限流   | 全系统 API ≤ 10,000 QPS（通过 Nginx 层） |
| 租户限流   | 单租户 ≤ 1,000 QPS                       |
| 用户限流   | 单用户 ≤ 100 QPS                         |
| 登录限流   | 单 IP 登录 ≤ 10 次/分钟                  |
| 请求体上限 | 通用 API ≤ 10MB，导入接口 ≤ 100MB        |
| 安全头     | 全面配置 HSTS、CSP、X-Frame-Options 等   |
| CORS       | 白名单域名，不允许 `*`                   |

## 2. 速率限制

### 2.1 三级限流策略

| 级别   | 维度      | 限流阈值   | 时间窗口 | 实现方式             |
| ------ | --------- | ---------- | -------- | -------------------- |
| 全局   | 全系统    | 10,000 QPS | 1 秒     | Nginx `limit_req`    |
| 租户级 | tenant_id | 1,000 QPS  | 1 秒     | Redis + 拦截器       |
| 用户级 | user_id   | 100 QPS    | 1 秒     | Redis + 拦截器       |
| 登录级 | IP 地址   | 10 次/分钟 | 1 分钟   | Redis + 登录接口拦截 |

### 2.2 限流实现

**Nginx 全局限流**：

```nginx
## 全局限流区域：10,000 req/s
limit_req_zone $binary_remote_addr zone=global:10m rate=10000r/s;

server {
    location /api/ {
        limit_req zone=global burst=2000 nodelay;
        proxy_pass http://backend;
    }
}
```

**应用层租户/用户限流（Redis 滑动窗口）**：

```java
@Component
public class RateLimitInterceptor implements HandlerInterceptor {

    private final StringRedisTemplate redisTemplate;

    @Override
    public boolean preHandle(HttpServletRequest request, ...) {
        // 用户级限流
        Long userId = UserContext.getUserId();
        if (userId != null && !checkRateLimit("user:" + userId, 100, 1)) {
            throw new BusinessException(ErrorCode.RATE_LIMIT_EXCEEDED);
        }

        // 租户级限流
        Long tenantId = TenantContext.getTenantId();
        if (tenantId != null && !checkRateLimit("tenant:" + tenantId, 1000, 1)) {
            throw new BusinessException(ErrorCode.RATE_LIMIT_EXCEEDED);
        }

        return true;
    }

    private boolean checkRateLimit(String key, int maxRequests, int windowSeconds) {
        String redisKey = "ratelimit:" + key;
        Long count = redisTemplate.opsForValue().increment(redisKey);
        if (count == 1) {
            redisTemplate.expire(redisKey, windowSeconds, TimeUnit.SECONDS);
        }
        return count <= maxRequests;
    }
}
```

### 2.3 限流响应

```json
// HTTP 429 Too Many Requests
{
  "code": 42900,
  "message": "请求过于频繁，请稍后重试",
  "data": {
    "retryAfter": 5,
    "limit": 100,
    "window": "1s"
  }
}
```

响应头：`Retry-After: 5`、`X-RateLimit-Limit: 100`、`X-RateLimit-Remaining: 0`

## 3. 请求安全

### 3.1 请求体大小限制

| 接口类型     | 最大请求体 | 配置位置                                       |
| ------------ | ---------- | ---------------------------------------------- |
| 通用 API     | 10 MB      | Spring `server.tomcat.max-http-form-post-size` |
| 文件导入     | 100 MB     | Controller 方法级注解                          |
| 通用文件上传 | 50 MB      | MultipartResolver                              |
| Nginx 全局   | 150 MB     | `client_max_body_size`                         |

```yaml
## application.yml
server:
  tomcat:
    max-http-form-post-size: 10MB

spring:
  servlet:
    multipart:
      max-file-size: 100MB
      max-request-size: 100MB
```

### 3.2 Content-Type 校验

```java
// Controller 方法必须明确声明 Content-Type
@PostMapping(value = "/reports", consumes = MediaType.APPLICATION_JSON_VALUE)
public Result<Report> createReport(@Valid @RequestBody ReportCreateReq req) { ... }

@PostMapping(value = "/import", consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
public Result<Task> importFile(@RequestPart MultipartFile file) { ... }
```

不匹配的 Content-Type 返回 HTTP 415。

### 3.3 参数校验

使用 Bean Validation（`@Valid` + 注解），在 Controller 层拦截：

```java
public class ReportCreateReq {
    @NotBlank @Size(max = 200)
    private String name;

    @NotNull
    private ReportTypeEnum type;

    @Size(max = 2000)
    private String description;

    @Pattern(regexp = "^[a-zA-Z0-9_-]{1,50}$")
    private String code;
}

// 全局校验异常处理
@ExceptionHandler(MethodArgumentNotValidException.class)
public Result<Void> handleValidation(MethodArgumentNotValidException e) {
    return Result.error(ErrorCode.PARAM_ERROR, e.getBindingResult().getFieldError().getDefaultMessage());
}
```

## 4. 防注入攻击

### 4.1 SQL 注入防护

| 防护层         | 策略                                                  |
| -------------- | ----------------------------------------------------- |
| MyBatis        | 全部使用 `#{}` 参数化查询，禁止 `${}`（除非明确审批） |
| DAO 层审查     | Code Review 强制检查 `${}` 使用，需审批豁免           |
| 数据库账号权限 | APP 账号仅 SELECT/INSERT/UPDATE/DELETE，无 DDL 权限   |
| 存储过程       | 禁止使用动态 SQL 拼接的存储过程                       |

```java
// ❌ 禁止
@Select("SELECT * FROM report_metadata WHERE name LIKE '%${keyword}%'")
List<Report> searchByKeyword(String keyword);

// ✅ 正确
@Select("SELECT * FROM report_metadata WHERE name LIKE CONCAT('%', #{keyword}, '%')")
List<Report> searchByKeyword(String keyword);
```

### 4.2 XSS 防护

| 防护层       | 策略                                     |
| ------------ | ---------------------------------------- |
| 后端输出     | JSON 序列化自动转义特殊字符              |
| 前端输入     | 输入框组件使用 PrimeReact 内置 HTML 转义 |
| 后端输入校验 | 对所有用户输入进行 HTML 标签过滤         |
| CSP 头       | 限制内联脚本和外部脚本来源               |

```java
// 通用 XSS 过滤
public class XssUtils {
    public static String clean(String input) {
        return input
            .replace("<", "&lt;")
            .replace(">", "&gt;")
            .replace("\"", "&quot;")
            .replace("'", "&#x27;")
            .replace("&", "&amp;");
    }
}
```

### 4.3 路径遍历防护

```java
// 文件操作时校验路径不包含 "../" 或 "..\\"
public void validateFilePath(String path) {
    if (path.contains("..") || path.contains("~")) {
        throw new BusinessException(ErrorCode.PARAM_ERROR, "非法文件路径");
    }
    // 构建绝对路径后再次校验是否在允许的目录范围内
    Path resolved = BASE_DIR.resolve(path).normalize().toAbsolutePath();
    if (!resolved.startsWith(BASE_DIR)) {
        throw new BusinessException(ErrorCode.PARAM_ERROR, "路径越界");
    }
}
```

## 5. CORS 与 CSRF

### 5.1 CORS 策略

```java
@Configuration
public class CorsConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
            .allowedOrigins("https://jinshu.example.com")  // 白名单，不允许 *
            .allowedMethods("GET", "POST", "PUT", "DELETE", "PATCH")
            .allowedHeaders("*")
            .allowCredentials(true)
            .maxAge(3600);
    }
}
```

### 5.2 CSRF 防护

- API 使用 JWT Bearer Token 认证（Token 不在 Cookie 中），天然防御 CSRF
- 如需要 Cookie 存储 Token，则设置 `SameSite=Strict`
- 状态变更接口（POST/PUT/DELETE）强制检查 JWT + 租户上下文

## 6. HTTP 安全头

```yaml
## Nginx 或 Spring Security 配置
headers:
  # 强制 HTTPS（HSTS）
  Strict-Transport-Security: "max-age=31536000; includeSubDomains; preload"
  # 禁止 MIME 嗅探
  X-Content-Type-Options: "nosniff"
  # 禁止嵌入 iframe（防 Clickjacking）
  X-Frame-Options: "DENY"
  # XSS 过滤器
  X-XSS-Protection: "0" # 现代浏览器已废弃，由 CSP 替代
  # 内容安全策略
  Content-Security-Policy: >
    default-src 'self';
    script-src 'self' 'unsafe-inline';
    style-src 'self' 'unsafe-inline';
    img-src 'self' data: blob:;
    font-src 'self';
    connect-src 'self' https://api.jinshu.example.com;
    frame-ancestors 'none';
    form-action 'self';
  # 引用策略
  Referrer-Policy: "strict-origin-when-cross-origin"
  # 权限策略
  Permissions-Policy: "camera=(), microphone=(), geolocation=()"
```

## 7. 文件上传安全

### 7.1 文件类型白名单

| 场景     | 白名单格式                                  |
| -------- | ------------------------------------------- |
| 报表导入 | `.xlsx`、`.xls`、`.csv`                     |
| 图片上传 | `.jpg`、`.jpeg`、`.png`、`.svg`（限制尺寸） |
| 模板上传 | `.json`、`.yaml`                            |

校验方式（魔数检查，非仅文件扩展名）：

```java
public boolean isValidExcel(MultipartFile file) {
    byte[] header = new byte[4];
    file.getInputStream().read(header);

    // XLSX 文件头：PK.. (ZIP)，XLS 文件头：D0 CF 11 E0
    String hex = HexFormat.of().formatHex(header);
    return hex.startsWith("504b") || hex.startsWith("d0cf11e0");
}
```

### 7.2 文件大小限制

参见 3.1 节。

### 7.3 文件内容校验

- 文件名：过滤 `../`、`~` 等路径遍历字符
- 文件名长度：不超过 255 字符
- 上传后病毒扫描：调用 ClamAV 扫描（可选，二期）

## 8. DDoS 防护

| 层级    | 策略                             |
| ------- | -------------------------------- |
| CDN/WAF | 云平台 DDoS 高防（生产环境建议） |
| Nginx   | `limit_conn` 限制并发连接数      |
| Nginx   | `limit_req` 限制请求速率         |
| 应用层  | Redis 滑动窗口限流（见第 2 节）  |
| K8s HPA | 自动水平扩展应对流量突增         |

## 9. 异常处理

| 攻击类型           | 检测方式                   | 响应                       |
| ------------------ | -------------------------- | -------------------------- |
| 暴力破解登录       | 单 IP 登录失败 ≥ 10次/分钟 | 返回 429 + 封禁 IP 30 分钟 |
| SQL 注入探测       | WAF 规则 / 敏感字符        | 返回 403，记录安全事件     |
| 路径遍历探测       | 请求路径含 `../`           | 返回 400                   |
| 超大数据包         | 请求体超过限制             | 返回 413                   |
| 不支持的 HTTP 方法 | OPTIONS/TRACE 等           | 返回 405                   |

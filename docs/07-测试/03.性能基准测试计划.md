## 1. 性能目标

### 1.1 核心性能指标（SLA）

| 场景                    | 指标要求 | 优先级 |
| ----------------------- | -------- | ------ |
| 小报表导入（< 1 万行）  | < 3 秒   | P0     |
| 中等报表导入（10 万行） | < 30 秒  | P0     |
| 大报表导入（100 万行）  | < 5 分钟 | P0     |
| 简单 PDF 生成（10 页）  | < 5 秒   | P0     |
| 复杂 PDF 生成（100 页） | < 30 秒  | P0     |
| 单条简单查询            | < 10ms   | P1     |
| 列表查询（分页 20 条）  | < 50ms   | P1     |
| 任务调度响应时间        | < 1 秒   | P1     |

### 1.2 并发与吞吐量

| 指标               | 要求     |
| ------------------ | -------- |
| 最大并发导入任务数 | 50       |
| 系统整体 QPS       | ≥ 100    |
| 单租户并发限制     | 5 个任务 |
| 95 线响应时间      | < 500ms  |
| 99 线响应时间      | < 2 秒   |

## 3. 基准测试用例

### 3.1 报表导入性能测试

#### TC-IMP-001: 小文件导入基准

| 项       | 值           |
| -------- | ------------ |
| 文件大小 | 1 万行，~1MB |
| 并发数   | 1            |
| 执行次数 | 5 次取平均   |
| 合格线   | < 3 秒       |

```
测试步骤：
1. 清理测试数据库
2. 上传 1 万行标准测试文件
3. 等待任务完成
4. 记录总耗时、内存峰值、CPU 峰值
5. 重复 5 次计算平均值
```

#### TC-IMP-002: 百万行导入基准

| 项       | 值                                 |
| -------- | ---------------------------------- |
| 文件大小 | 100 万行，~50MB                    |
| 并发数   | 1                                  |
| 执行次数 | 3 次取平均                         |
| 合格线   | < 5 分钟                           |
| 内存限制 | JVM 堆内存 < 1GB（流式读取防 OOM） |

```
关键监控点：
- JVM 堆内存使用曲线（不能持续增长）
- GC 次数和停顿时间
- 数据库连接池使用率
- 磁盘 IO 吞吐量
```

#### TC-IMP-003: 多租户并发导入

| 项           | 值                                 |
| ------------ | ---------------------------------- |
| 租户数量     | 10 个租户                          |
| 每个租户文件 | 1 万行                             |
| 并发模式     | 同时发起导入                       |
| 合格线       | 所有任务在 15 秒内完成，无数据泄露 |

```
验证点：
1. 公平调度：租户之间任务完成时间差异 < 2x
2. 数据隔离：每个租户只能看到自己的任务
3. 无资源耗尽：数据库连接不耗尽，MQ 不堆积
```

## 4. 测试工具

### 4.1 负载测试工具

| 工具         | 用途               |
| ------------ | ------------------ |
| JMeter       | API 并发压测       |
| k6           | 现代化 JS 脚本压测 |
| Apache Bench | 简单 HTTP 压测     |

### 4.2 监控工具

| 维度     | 工具                             |
| -------- | -------------------------------- |
| JVM 指标 | JConsole / VisualVM / Micrometer |
| 数据库   | pg_stat_statements 扩展          |
| 系统资源 | Prometheus + Grafana             |
| 应用性能 | Arthas（阿里巴巴）               |

### 4.3 关键监控指标

```
JVM:
  - 堆内存使用率
  - GC 次数和停顿时间
  - 线程数
  - 类加载数

数据库:
  - 连接池使用率
  - 慢查询数（> 100ms）
  - 锁等待时间
  - 事务提交速率

系统:
  - CPU 使用率（用户态 + 内核态）
  - 内存使用率
  - 磁盘 IO (r/w)
  - 网络吞吐量
```

## 5. 性能回归测试

### 5.1 基线建立

每次发布前，运行核心性能测试用例，与基线对比：

```
性能退化阈值：
  - P0 指标：退化 > 10% → 阻断发布
  - P1 指标：退化 > 20% → 告警，记录说明
```

### 5.2 CI 集成

```yaml
# GitHub Actions 示例
performance-test:
  runs-on: performance-runner
  steps:
    - name: Run import benchmark
      run: ./mvnw jmh:benchmark -Djmh.includes=ImportBenchmark

    - name: Compare with baseline
      run: |
        python scripts/compare_perf.py \
          --baseline=baseline/import_10k.json \
          --current=results/import_10k.json \
          --threshold=10%
```

## 6. 稳定性测试

### 6.1 长时间运行测试

| 项     | 值                               |
| ------ | -------------------------------- |
| 时长   | 72 小时                          |
| 负载   | 30% 峰值 QPS                     |
| 合格线 | 内存不泄漏，无 OOM，响应时间稳定 |

### 6.2 故障注入测试

| 故障场景            | 验证点                 |
| ------------------- | ---------------------- |
| 数据库重启          | 任务不丢失，自动重试   |
| MQ 宕机             | 消息不丢失，恢复后重发 |
| 应用重启            | 进行中的任务可恢复     |
| 网络抖动（丢包 5%） | 性能平滑降级，无雪崩   |

## 7. 测试报告模板

每次性能测试输出以下内容：

```
=== 性能测试报告 ===

测试日期: 2026-xx-xx
测试版本: git commit hash
环境配置: 8C32G, PostgreSQL 15

--- 测试结果摘要 ---
| 用例 | 平均值 | 最大值 | 最小值 | 基线 | 偏差 | 结论 |
|------|-------|-------|-------|------|------|------|
| TC-IMP-001 | 2.1s | 2.5s | 1.8s | 2.0s | +5% | ✅ 通过 |
| TC-IMP-002 | 4m30s | 4m45s | 4m20s | 5m | -10% | ✅ 通过 |

--- JVM 指标 ---
堆内存峰值: 850MB (目标 < 1GB) ✅
Full GC 次数: 0 次 ✅
平均停顿: < 10ms ✅

--- 数据库指标 ---
慢查询数: 2 条 (需要优化) ⚠️
平均事务响应: 8ms ✅

--- 结论 ---
总体性能达标，2 条慢查询需要在发布前优化。
```

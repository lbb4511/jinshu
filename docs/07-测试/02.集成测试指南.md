## 1. 集成测试定义与范围

### 1.1 什么是集成测试

集成测试验证**多个组件协同工作**时的正确性，范围包括：

- Service 层 + DAO 层 + 真实数据库
- Controller 层 + Service 层完整调用链
- 消息队列生产者 + 消费者
- 第三方 API 调用（文件存储、邮件推送等）

### 1.2 与单元测试的区别

| 维度     | 单元测试       | 集成测试                  |
| -------- | -------------- | ------------------------- |
| 范围     | 单个类/方法    | 多层组件协同              |
| 外部依赖 | Mock 替换      | 真实组件或 Testcontainers |
| 执行速度 | < 100ms        | 几秒到几十秒              |
| 数据库   | 不连或 H2 内存 | 真实 PostgreSQL 15        |

## 2. 测试环境与工具

### 2.1 Testcontainers 容器化依赖

所有外部依赖通过 Docker 容器启动，保证环境一致：

```java
@Testcontainers
@SpringBootTest
public class ReportImportIntegrationTest {

    @Container
    static PostgreSQLContainer<?> postgres =
        new PostgreSQLContainer<>("postgres:15-alpine")
            .withDatabaseName("jinshu_test")
            .withUsername("test")
            .withPassword("test");

    @Container
    static RabbitMQContainer rabbitmq =
        new RabbitMQContainer("rabbitmq:3.12-management-alpine");

    @DynamicPropertySource
    static void properties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", postgres::getJdbcUrl);
        registry.add("spring.rabbitmq.host", rabbitmq::getHost);
    }
}
```

### 2.2 数据库初始化策略

| 策略          | 适用场景       | 实现方式                   |
| ------------- | -------------- | -------------------------- |
| `create-drop` | 单测/集成测试  | Hibernate/JPA 自动建表删表 |
| `schema.sql`  | 需要精确表结构 | Flyway 迁移脚本            |
| 测试数据导入  | 需要预置数据   | `@Sql` 注解执行 DML        |

## 3. 集成测试分类

### 3.1 数据访问层集成测试（DAO + DB）

**目标**：验证 SQL、事务、索引、多租户过滤是否正确

```java
@SpringBootTest
@Transactional  // 每个测试后自动回滚
class TaskDAOTest {

    @Autowired
    private TaskDAO taskDAO;

    @BeforeEach
    void setUp() {
        TenantContext.setCurrentTenant(1L);
    }

    @Test
    void given_tenantData_when_queryOtherTenant_then_returnEmpty() {
        // Given - 插入租户 1 的数据
        Task task = createTask(1L, "report1.xlsx");
        taskDAO.insert(task);

        // When - 切换到租户 2 查询
        TenantContext.setCurrentTenant(2L);
        List<Task> tasks = taskDAO.listAll();

        // Then - 租户 2 看不到租户 1 的数据
        assertThat(tasks).isEmpty();  // ✅ 多租户隔离生效
    }

    @Test
    void given_skipTenantFilter_when_query_then_returnAllTenants() {
        // Given
        taskDAO.insert(createTask(1L, "t1.xlsx"));
        taskDAO.insert(createTask(2L, "t2.xlsx"));

        // When - 用超级管理员权限查询
        try (var ignored = TenantContext.withSkipFilter()) {
            List<Task> all = taskDAO.listAll();

            // Then
            assertThat(all).hasSize(2);
        }
    }
}
```

### 3.2 业务服务层集成测试（Service + DAO + DB）

**目标**：验证完整业务逻辑，包括事务、异常传播

```java
@SpringBootTest
@Transactional
class ReportImportServiceTest {

    @Autowired
    private ReportImportService importService;

    @Autowired
    private TaskDAO taskDAO;

    @Test
    void given_largeFile_when_import_then_createTaskAndSendMessage() {
        // Given
        ReportImportDTO dto = ReportImportDTO.builder()
            .fileName("million_rows.xlsx")
            .fileSize(50 * 1024 * 1024L)  // 50MB
            .build();

        // When
        Result<TaskVO> result = importService.importReport(dto);

        // Then - 验证数据库状态
        assertThat(result.isSuccess()).isTrue();
        Task task = taskDAO.selectById(result.getData().getTaskId());
        assertThat(task.getStatus()).isEqualTo(TaskStatus.PENDING);

        // Then - 验证消息已发送（通过 Testcontainers 检查队列）
        Integer messageCount = rabbitmqAdmin.getQueueMessageCount("import.queue");
        assertThat(messageCount).isEqualTo(1);
    }

    @Test
    void given_invalidFile_when_import_then_rollbackTransaction() {
        // Given
        int beforeCount = taskDAO.countByTenant(1L);

        ReportImportDTO invalidDto = ReportImportDTO.builder()
            .fileName("corrupted.xlsx")
            .build();

        // When - 导入会抛出异常
        assertThatThrownBy(() -> importService.importReport(invalidDto))
            .isInstanceOf(FileParseException.class);

        // Then - 事务回滚，数据库没有脏数据
        int afterCount = taskDAO.countByTenant(1L);
        assertThat(afterCount).isEqualTo(beforeCount);  // ✅ 回滚正确
    }
}
```

### 3.3 API 层集成测试（Controller + Service + 全链路）

**目标**：验证 HTTP 请求/响应、参数校验、权限拦截、全局异常处理

```java
@SpringBootTest
@AutoConfigureMockMvc
class ReportImportControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @Test
    @WithMockUser(roles = "USER")
    void given_authenticatedUser_when_import_then_return200() throws Exception {
        // Given
        MockMultipartFile file = new MockMultipartFile(
            "file", "report.xlsx",
            "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
            Files.readAllBytes(Paths.get("test-data/excel/valid_100_rows.xlsx"))
        );

        // When + Then
        mockMvc.perform(multipart("/api/v1/report/import")
                .file(file)
                .header("X-Tenant-Id", "1"))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.code").value(0))
            .andExpect(jsonPath("$.data.taskId").isNotEmpty());
    }

    @Test
    void given_noTenantHeader_when_import_then_return400() throws Exception {
        mockMvc.perform(multipart("/api/v1/report/import")
                .file(createValidFile()))
            .andExpect(status().isBadRequest())
            .andExpect(jsonPath("$.code").value(ErrorCode.TENANT_REQUIRED.getCode()));
    }

    @Test
    void given_fileTooLarge_when_import_then_return413() throws Exception {
        // 100MB 超大文件
        MockMultipartFile largeFile = createLargeFile(100 * 1024 * 1024);

        mockMvc.perform(multipart("/api/v1/report/import")
                .file(largeFile)
                .header("X-Tenant-Id", "1"))
            .andExpect(status().isPayloadTooLarge());
    }
}
```

```
test-data/
├── excel/
│   ├── valid_100_rows.xlsx       # 正常小文件
│   ├── valid_10000_rows.xlsx     # 正常中等文件
│   ├── valid_1000000_rows.xlsx   # 百万行大文件
│   ├── empty_file.xlsx           # 空文件
│   ├── corrupted.xlsx            # 损坏文件
│   ├── wrong_format.pdf          # 格式错误
│   └── special_chars.xlsx        # 特殊字符边界
├── sql/
│   ├── tenant_1_basic_data.sql   # 租户 1 基础数据

    └── sample_10_pages.pdf       # PDF 渲染测试
```

### 4.2 测试数据原则

1. **真实但脱敏**：数据格式与生产一致，但无真实敏感信息
2. **覆盖边界**：空文件、超大文件、特殊字符、异常格式
3. **版本控制**：测试数据文件纳入 Git 管理（大文件用 LFS）

## 6. 并发集成测试

### 6.1 多租户并发导入测试

```java
@Test
void given_10Tenants_concurrentImport_then_noDataLeakage()
    throws InterruptedException {

    ExecutorService executor = Executors.newFixedThreadPool(10);
    CountDownLatch latch = new CountDownLatch(10);

    List<Future<Long>> results = new ArrayList<>();

    // 10 个租户同时导入
    for (long tenantId = 1; tenantId <= 10; tenantId++) {
        long tid = tenantId;
        results.add(executor.submit(() -> {
            try {
                TenantContext.setCurrentTenant(tid);
                Result<TaskVO> result = importService.importReport(createDto());
                return result.getData().getTaskId();
            } finally {
                latch.countDown();
            }
        }));
    }

    latch.await(30, TimeUnit.SECONDS);
    executor.shutdown();

    // 验证：每个租户只能看到自己的任务
    for (long tenantId = 1; tenantId <= 10; tenantId++) {
        TenantContext.setCurrentTenant(tenantId);
        List<Task> tasks = taskDAO.listAll();
        assertThat(tasks).hasSize(1);  // 每个租户只有 1 条记录
        assertThat(tasks.get(0).getTenantId()).isEqualTo(tenantId);
    }
}
```

## 7. 集成测试最佳实践

### 7.1 测试隔离

✅ **每个测试独立，不依赖其他测试的副作用**

- 使用 `@Transactional` 自动回滚
- 或在 `@AfterEach` 中手动清理数据
- 绝不假设数据库初始状态

### 7.2 避免过度使用 SpringBootTest

- DAO 层测试：`@MybatisTest` 更轻量
- Controller 层测试：`@WebMvcTest` + Mock Service
- 全链路测试：才用 `@SpringBootTest`

### 7.3 测试命名

集成测试方法名建议加上场景描述：

```java
@Test
void integration_given_multiTenantData_when_query_withoutSwitch_then_isolation() {}
```

| 测试类型       | 触发时机     | 超时    | 失败处理     |
| -------------- | ------------ | ------- | ------------ |
| 单元测试       | 每次 PR 提交 | 5 分钟  | 立即阻断合并 |
| 集成测试       | 每次 PR 提交 | 30 分钟 | 立即阻断合并 |

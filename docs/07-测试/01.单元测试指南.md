## 1. 测试框架选型

### 1.1 后端 Java 测试栈

| 组件      | 选型        | 说明                       |
| --------- | ----------- | -------------------------- |
| 测试框架  | JUnit 5     | Jupiter 引擎，支持 Java 17 |
| Mock 框架 | Mockito 4.x | 轻量、高性能               |
| 断言库    | AssertJ     | 流式断言，可读性好         |
| JSON 断言 | JSONAssert  | 复杂 JSON 结构对比         |

### 1.2 前端 React 测试栈

| 组件     | 选型                      | 说明                 |
| -------- | ------------------------- | -------------------- |
| 测试框架 | Jest                      | Facebook 官方推荐    |
| 组件测试 | React Testing Library     | 关注用户行为而非实现 |
| 断言库   | @testing-library/jest-dom | DOM 专用断言         |

## 2. 测试覆盖要求

### 2.1 覆盖率指标

| 模块类型            | 行覆盖率 | 分支覆盖率 | 说明                     |
| ------------------- | -------- | ---------- | ------------------------ |
| 核心业务（Service） | ≥ 80%    | ≥ 70%      | 导入、导出、审批核心逻辑 |
| 工具类/公共组件     | ≥ 90%    | ≥ 85%      | 异常分支必须全覆盖       |
| DAO/Mapper          | ≥ 70%    | ≥ 60%      | 复杂 SQL 需单独测试      |
| Controller          | ≥ 60%    | ≥ 50%      | 参数校验、异常转换逻辑   |

### 2.2 必须覆盖的场景

```
□ 正常流程：输入合法，输出符合预期
□ 边界值：最小值、最大值、零值、空值
□ 异常流程：参数非法、依赖失败、超时
□ 多租户隔离：跨租户数据访问必须被拦截
□ 权限校验：未登录/无权限必须返回 401/403
□ 并发场景：竞态条件、死锁检测
```

## 3. 测试命名规范

### 3.1 测试类命名

```
{被测试类名}Test.java
```

示例：

- `ReportImportServiceTest.java`
- `TenantInterceptorTest.java`

### 3.2 测试方法命名

采用 **Given-When-Then** 三段式：

```java
@Test
void given_条件描述_when_操作描述_then_预期结果() {
    // 测试代码
}
```

✅ 好的示例：

```java
@Test
void given_validExcelFile_when_import_then_returnSuccessWithTaskId() {}

@Test
void given_tenantUser_when_queryOtherTenantData_then_throwAccessDenied() {}

@Test
void given_emptyFile_when_import_then_throwInvalidFileException() {}
```

❌ 坏的示例：

```java
@Test
void testImport() {}  // 太模糊

@Test
void test1() {}       // 无意义编号
```

## 4. 测试结构规范

### 4.1 标准三段式结构

每个测试方法必须清晰分为三部分：

```java
@Test
void given_validInput_when_operation_then_expectedResult() {
    // 1. Given - 准备测试数据
    TenantContext.setCurrentTenant(1L);
    ReportImportDTO dto = createValidImportDTO();

    // 2. When - 执行被测试操作
    Result<TaskVO> result = reportImportService.importReport(dto);

    // 3. Then - 断言结果
    assertThat(result.isSuccess()).isTrue();
    assertThat(result.getData().getTaskId()).isNotNull();
}
```

### 4.2 测试数据管理

#### 方式一：@BeforeEach 通用数据

```java
@BeforeEach
void setUp() {
    // 每个测试前都执行的通用初始化
    TenantContext.setCurrentTenant(DEFAULT_TENANT_ID);
}

@AfterEach
void tearDown() {
    // 每个测试后清理
    TenantContext.clear();
}
```

#### 方式二：辅助方法 + 建造者模式

```java
private ReportImportDTO createValidImportDTO() {
    return ReportImportDTO.builder()
        .fileName("test.xlsx")
        .fileSize(102400L)
        .build();
}
```

## 5. Mock 使用规范

### 5.1 什么该 Mock，什么不该

| 对象                 | 是否 Mock | 理由                                   |
| -------------------- | --------- | -------------------------------------- |
| 外部服务（HTTP、MQ） | ✅ 是     | 避免网络依赖，稳定可控                 |
| 数据库（DAO层）      | ❌ 否     | 使用 Testcontainers 测真实数据库       |
| 其他 Service         | ⚠️ 看情况 | 被依赖 Service 本身已充分测试时可 Mock |
| 工具类（静态方法）   | ❌ 否     | 直接调用，测真实逻辑                   |

### 5.2 Mock 示例

```java
@ExtendWith(MockitoExtension.class)
class ReportImportServiceTest {

    @Mock
    private RabbitTemplate rabbitTemplate;  // 外部依赖 Mock

    @InjectMocks
    private ReportImportService reportImportService;

    @Test
    void given_validTask_when_submit_then_sendMessage() {
        // Given
        Task task = createValidTask();

        // When
        reportImportService.submitTask(task);

        // Then
        verify(rabbitTemplate, times(1))
            .convertAndSend(eq("import.queue"), any(TaskMessage.class));
    }
}
```

### 5.3 禁止过度 Mock

❌ 反模式：把被测试类本身的方法也 Mock 了

```java
// 绝对禁止！这样测试的是 Mock 不是真实代码
@Spy
@InjectMocks
private ReportImportService service;

doReturn(mockData).when(service).someInternalMethod();
```

## 6. 断言规范

### 6.1 使用 AssertJ 流式断言

✅ 推荐：

```java
assertThat(result.getCode()).isEqualTo(ErrorCode.SUCCESS.getCode());
assertThat(result.getData()).isNotNull();
assertThat(result.getData().getRows()).hasSize(100);
```

❌ 不推荐（JUnit 原生断言）：

```java
assertEquals(ErrorCode.SUCCESS.getCode(), result.getCode());
assertNotNull(result.getData());
assertEquals(100, result.getData().getRows().size());
```

### 6.2 异常断言

```java
// 方式一：assertThatThrownBy
assertThatThrownBy(() -> service.importReport(invalidDTO))
    .isInstanceOf(InvalidFileException.class)
    .hasMessageContaining("文件格式不支持");

// 方式二：assertThatExceptionOfType
assertThatExceptionOfType(AccessDeniedException.class)
    .isThrownBy(() -> service.queryOtherTenantData(999L));
```

## 7. 测试分类与 Tag

### 7.1 常用 Tag

```java
@Test
@Tag("fast")       // 快速测试，CI 每次运行
@Tag("slow")       // 慢速测试， nightly 构建运行
@Tag("security")   // 安全相关测试，安全扫描时运行
@Tag("tenant")     // 多租户相关测试
```

### 7.2 执行分组配置

`build.gradle` 或 `pom.xml` 中配置：

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-surefire-plugin</artifactId>
    <configuration>
        <groups>fast</groups>          <!-- CI 默认只跑 fast -->
        <excludedGroups>slow</excludedGroups>
    </configuration>
</plugin>
```

## 8. 参数化测试

### 8.1 多组输入用同一份测试逻辑

```java
@ParameterizedTest
@ValueSource(strings = {"test.xlsx", "test.xls", "test.csv"})
void given_supportedFormat_when_validate_then_pass(String fileName) {
    assertThat(FileValidator.validate(fileName)).isTrue();
}

@ParameterizedTest
@CsvSource({
    "0, false",
    "1, true",
    "1000000, true",
    "1000001, false"
})
void given_rowCount_when_validate_then_expected(long rows, boolean expected) {
    assertThat(RowValidator.validate(rows)).isEqualTo(expected);
}
```

## 9. 测试性能要求

### 9.1 单测执行速度

| 标签           | 单测最大耗时 | 说明              |
| -------------- | ------------ | ----------------- |
| `@Tag("fast")` | < 100ms      | 无 IO，纯内存计算 |
| `@Tag("slow")` | < 5s         | 含数据库、文件 IO |

### 9.2 超时自动失败

```java
@Test
@Timeout(value = 100, unit = TimeUnit.MILLISECONDS)
void given_smallFile_when_parse_then_completeWithinTime() {
    // 必须在 100ms 内完成，否则算失败
}
```

## 10. 可测试性设计原则

### 10.1 依赖注入而非硬编码

✅ 好的设计（可测试）：

```java
@Service
public class ReportImportService {
    // 依赖通过构造函数注入，测试时可替换
    private final TaskDAO taskDAO;
    private final FileParser fileParser;

    public ReportImportService(TaskDAO taskDAO, FileParser fileParser) {
        this.taskDAO = taskDAO;
        this.fileParser = fileParser;
    }
}
```

❌ 坏的设计（不可测试）：

```java
@Service
public class ReportImportService {
    // 硬编码 new 出来，无法替换
    private final TaskDAO taskDAO = new TaskDAOImpl();
    private final FileParser parser = EasyExcelParser.getInstance();
}
```

### 10.2 方法职责单一

每个方法只做一件事，圈复杂度 < 10。复杂方法拆分为多个小方法，每个小方法单独测试。

## 11. 测试反模式（禁止）

### ❌ 模式 1：随机测试数据

```java
// 禁止！测试失败不可复现
int randomRows = new Random().nextInt(1000);
```

### ❌ 模式 2：测试间相互依赖

```java
// 禁止！测试 2 依赖测试 1 的执行结果
@Test void test1_createData() { ... }
@Test void test2_useData() { ... }
```

### ❌ 模式 3：空断言 / 无意义断言

```java
// 禁止！等于没测
@Test void testImport() {
    service.importReport(dto);  // 没有任何断言！
}
```

### ❌ 模式 4：try-catch 吞异常

```java
// 禁止！异常被吃掉，测试永远"通过"
@Test void test() {
    try {
        service.someMethod();
    } catch (Exception e) {
        // 空 catch！
    }
}
```

### ❌ 模式 5：System.out.println 调试

```java
// 禁止！用日志或断言
System.out.println("result = " + result);
```

## 12. 好的单元测试的 F.I.R.S.T 原则

| 原则                | 说明                         |
| ------------------- | ---------------------------- |
| **F**ast            | 快速，毫秒级完成             |
| **I**ndependent     | 独立，测试间不互相依赖       |
| **R**epeatable      | 可重复，每次运行结果一致     |
| **S**elf-validating | 自验证，通过断言自动判断成败 |
| **T**imely          | 及时，编码前或编码同时写测试 |

## 13. 代码示例模板

```java
package com.jinshu.service;

import org.junit.jupiter.api.*;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;
import static org.assertj.core.api.Assertions.*;

@DisplayName("报表导入服务测试")
class ReportImportServiceTest {

    private ReportImportService service;

    @BeforeEach
    void setUp() {
        service = new ReportImportService();
        TenantContext.setCurrentTenant(1L);
    }

    @AfterEach
    void tearDown() {
        TenantContext.clear();
    }

    @Test
    @DisplayName("正常导入：合法Excel文件导入成功")
    void given_validExcelFile_when_import_then_returnSuccess() {
        // Given
        ReportImportDTO dto = ReportImportDTO.builder()
            .fileName("report.xlsx")
            .fileSize(102400L)
            .build();

        // When
        Result<TaskVO> result = service.importReport(dto);

        // Then
        assertThat(result.isSuccess()).isTrue();
        assertThat(result.getData().getTaskId()).isNotNull();
    }

    @Test
    @DisplayName("异常场景：空文件抛出异常")
    void given_emptyFile_when_import_then_throwException() {
        // Given
        ReportImportDTO dto = createEmptyFileDTO();

        // When + Then
        assertThatThrownBy(() -> service.importReport(dto))
            .isInstanceOf(InvalidFileException.class);
    }
}
```

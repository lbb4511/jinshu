## 核心安全原则

### 1. 默认为安全（Secure by Default）

- 不安全的配置默认关闭
- 默认权限最小化
- 新功能必须经过安全评审

### 2. 纵深防御（Defense in Depth）

- 多层防护：网络层 → 应用层 → 数据层
- 任何一层被突破都不会导致全局失控

### 3. 最小权限原则

- 每个用户/服务只拥有完成任务所需的最小权限
- 数据库账号只有 SELECT/INSERT，绝对不给 DROP/ALTER
- 管理后台必须走内网 + 双因素认证

### 4. 安全左移（Shift Left）

- 设计阶段就要考虑安全
- 代码提交前自动安全扫描
- 安全问题发现越早，修复成本越低

## 不可逾越的红线

以下行为**绝对禁止**，违者直接驱逐：

1. ❌ 在代码中硬编码密码、密钥、Token
2. ❌ 绕过多租户隔离直接查全量表
3. ❌ 删除或篡改审计日志
4. ❌ 敏感数据明文存储或明文打印到日志
5. ❌ 引入 GPL/商业代码而不申报
6. ❌ 不经评审直接在生产环境改数据
7. ❌ 把生产环境数据导出到本地/测试环境

## 安全事故响应流程

```
发现安全事故
    ↓
立即报告安全负责人（24小时响应）
    ↓
紧急止血（隔离受影响的系统/账号）
    ↓
排查影响范围 + 取证
    ↓
修复漏洞
    ↓
复盘 + 改进措施
    ↓
更新本文档
```

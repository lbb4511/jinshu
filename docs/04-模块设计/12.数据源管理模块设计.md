## 1. 背景与目标

### 1.1 问题背景

报表系统的核心输入是外部数据源。当前系统虽然已定义 `data_source` 表（DDL 中），但缺少完整的模块设计文档，导致以下问题：

| 问题                     | 严重度  | 影响                                           |
| ------------------------ | ------- | ---------------------------------------------- |
| 数据源管理模块无设计文档 | 🔴 致命 | 权限矩阵、审计规范、API 路径均预留但无具体设计 |
| 连接凭据存储安全未明确   | 🔴 高   | `connection_config` JSONB 中密码字段需加密     |
| 连接器扩展机制未定义     | 🟡 中   | 新增数据库类型需修改核心代码                   |
| 连接池策略未统一         | 🟡 中   | 多数据源并发连接可能导致资源耗尽               |

### 1.2 设计目标

| 指标       | 要求                                 |
| ---------- | ------------------------------------ |
| 数据源类型 | 初始支持 PostgreSQL、MySQL，可扩展   |
| 凭据安全   | 密码 100% 加密存储（AES-256-GCM）    |
| 连接测试   | 测试连接耗时 < 5s，超时则判定失败    |
| 多租户隔离 | 数据源仅创建租户可见                 |
| 审计覆盖   | 所有 CRUD 操作 100% 记录审计日志     |
| 连接池隔离 | 每个数据源独立连接池，最大 10 个连接 |

### 1.3 非目标

- 不在此模块实现数据同步功能（由 `data-sync/` 模块负责）
- 不实现跨数据源联邦查询（二期）
- 不实现数据源模板/市场功能（二期功能设计）
- 不实现实时监控数据源健康状态（由可观测性模块负责）

## 2. 需求分析

### 2.1 功能需求

- [x] **创建数据源**：配置连接类型、地址、端口、数据库名、凭据
- [x] **编辑数据源**：修改配置（密码修改需旧密码验证）
- [x] **删除数据源**：软删除，关联报表存在时阻止删除
- [x] **测试连接**：使用配置参数尝试建立连接，返回成功/失败及错误信息
- [x] **数据源列表**：支持按名称搜索、按类型筛选，分页
- [x] **数据预览**：查看数据源的表列表、表结构、前 100 行数据
- [x] **凭据加密**：connection_config 中密码字段自动加密存储
- [x] **审计记录**：所有操作自动记录到 audit_log

### 2.2 用户故事

```
作为 报表设计者，我希望 配置一个 PostgreSQL 数据源并测试连接成功，
以便 后续基于该数据源创建报表导入任务。

作为 租户管理员，我希望 查看本租户下所有数据源的连接状态，
以便 及时发现失效的数据源并更新配置。

作为 安全管理员，我希望 所有数据源凭据加密存储，且无人能查看明文密码，
以便 满足数据安全合规要求。
```

### 2.3 边界场景

| 场景                   | 处理方式                                                      |
| ---------------------- | ------------------------------------------------------------- |
| 数据源被报表引用时删除 | 返回错误"该数据源被 N 个报表引用，无法删除"，列出引用报表列表 |
| 连接测试超时           | 5 秒超时，返回"连接超时：请检查网络和防火墙配置"              |
| 同一租户下同名数据源   | 数据库 UNIQUE(tenant_id, name) 约束，创建时返回错误           |
| 密码字段回显           | 编辑时密码字段不回显，提供"修改密码"独立入口                  |
| 连接密码过期/变更后    | 关联的导入任务失败时记录错误日志，通知管理员更新凭据          |
| 未加密的历史数据源密码 | 迁移脚本检查并加密全部 `connection_config->>'password'`       |

## 3. 架构设计

### 3.1 整体流程

```
用户
  │
  ▼
DataSourceController  ──参数校验
  │
  ▼
DataSourceService  ──业务逻辑、凭据加密/解密
  │
  ├──► DataSourceMapper  ──数据库 CRUD
  │
  ├──► ConnectionTestService  ──测试连接（独立线程池，5s 超时）
  │
  └──► AuditLogger  ──审计日志记录
```

### 3.2 连接器架构

使用策略模式实现可扩展的连接器：

```java
// 连接器接口
public interface DataSourceConnector {
    DataSourceType getType();           // 支持的数据源类型
    boolean testConnection(DataSourceConfig config);  // 测试连接
    List<TableInfo> listTables(DataSourceConfig config);  // 表列表
    TableSchema getTableSchema(DataSourceConfig config, String tableName);  // 表结构
    Connection getConnection(DataSourceConfig config);  // 获取连接（供数据同步使用）
}

// 内置实现
PostgreSQLConnector implements DataSourceConnector
MySQLConnector implements DataSourceConnector
```

新数据源类型只需实现 `DataSourceConnector` 接口并注册到 Spring 容器即可扩展。

### 3.3 模块划分

| 模块       | 职责                   | 所属 Gradle 模块     |
| ---------- | ---------------------- | -------------------- |
| Controller | 参数校验、调用 Service | `api/`               |
| Service    | 业务逻辑、凭据加解密   | `api/`               |
| Connector  | 连接管理、元数据查询   | `api/`（可后续独立） |
| Mapper     | 数据库 CRUD            | `api/`               |

## 4. 数据库设计

### 4.1 data_source 表

现有 DDL（`sql/ddl/01_init_schema.sql`）已定义，模块设计补充说明：

| 字段                | 类型            | 说明                                    |
| ------------------- | --------------- | --------------------------------------- |
| `id`                | BIGSERIAL PK    | 主键                                    |
| `tenant_id`         | BIGINT NOT NULL | 租户 ID                                 |
| `name`              | VARCHAR(200)    | 数据源名称（租户内唯一）                |
| `type`              | VARCHAR(50)     | 类型：POSTGRESQL / MYSQL                |
| `host`              | VARCHAR(255)    | 主机地址                                |
| `port`              | INTEGER         | 端口号                                  |
| `database_name`     | VARCHAR(200)    | 数据库名称                              |
| `username`          | VARCHAR(100)    | 连接用户名                              |
| `connection_config` | JSONB           | 连接配置（含加密密码、SSL、连接池参数） |
| `status`            | VARCHAR(20)     | 状态：ACTIVE / INACTIVE / ERROR         |
| `last_test_time`    | TIMESTAMP       | 最近一次连接测试时间                    |
| `last_test_result`  | VARCHAR(500)    | 最近一次连接测试结果                    |
| `description`       | TEXT            | 描述                                    |
| `created_by`        | BIGINT          | 创建人                                  |
| `updated_by`        | BIGINT          | 最后修改人                              |
| `created_at`        | TIMESTAMP       | 创建时间                                |
| `updated_at`        | TIMESTAMP       | 更新时间                                |
| `deleted_at`        | TIMESTAMP       | 软删除时间                              |

**connection_config JSONB 结构**：

```json
{
  "password": "AES256:iv:encrypted_data",
  "ssl_enabled": true,
  "ssl_mode": "require",
  "pool_min_idle": 2,
  "pool_max_active": 10,
  "connect_timeout": 5000,
  "socket_timeout": 30000,
  "extra_params": {}
}
```

### 4.2 data_source_connection_log 表（新增）

记录每次连接测试的结果，用于排查连接问题：

| 字段               | 类型            | 说明                   |
| ------------------ | --------------- | ---------------------- |
| `id`               | BIGSERIAL PK    | 主键                   |
| `tenant_id`        | BIGINT NOT NULL | 租户 ID                |
| `data_source_id`   | BIGINT NOT NULL | 数据源 ID              |
| `test_user_id`     | BIGINT          | 测试人用户 ID          |
| `success`          | BOOLEAN         | 是否成功               |
| `error_message`    | TEXT            | 错误信息（失败时记录） |
| `response_time_ms` | INTEGER         | 响应时间（毫秒）       |
| `created_at`       | TIMESTAMP       | 测试时间               |


## 6. 异常处理

| 场景                     | 错误码 | HTTP 状态码 | 处理方式                 |
| ------------------------ | ------ | ----------- | ------------------------ |
| 数据源名称重复           | 40900  | 409         | 返回"数据源名称已存在"   |
| 数据源不存在             | 40400  | 404         | 返回"数据源不存在"       |
| 数据源被报表引用无法删除 | 40901  | 409         | 返回引用报表数量与列表   |
| 连接测试失败             | 50200  | 502         | 返回具体错误原因         |
| 连接测试超时             | 50400  | 504         | 返回"连接超时"           |
| 密码解密失败             | 50001  | 500         | 记录异常，返回"凭据异常" |
| 数据预览超出行数         | 40001  | 400         | 限制最多返回 100 行      |
| 不支持的数据源类型       | 40002  | 400         | 返回"不支持的数据源类型" |

## 7. 性能考量

| 考量点       | 策略                                                   |
| ------------ | ------------------------------------------------------ |
| 连接测试并发 | 使用固定大小线程池（5 线程），避免大量连接测试占满资源 |
| 数据预览     | 限制只能返回 100 行，禁止 SELECT \* FROM 大表          |
| 表列表查询   | 从 information_schema 查询，异步缓存 5 分钟            |
| 连接池隔离   | 每个数据源独立 HikariCP，最大 10 连接，最少 2 空闲     |
| 凭据加解密   | AES-256-GCM 加解密 < 1ms，对接口响应无影响             |

## 8. 安全考量

### 8.1 多租户隔离

- 所有数据源操作强制带 `tenant_id` 过滤，不可跨租户访问
- `data_source_connection_log` 表也按租户隔离

### 8.2 凭据加密

- `connection_config` JSONB 中的 `password` 字段使用 AES-256-GCM 加密存储
- 密钥从环境变量 `JINSHU_ENCRYPTION_KEY` 读取（K8s Secret 注入）
- 密码明文从未出现在日志、API 响应、数据库普通字段中
- 编辑密码需先验证旧密码（可选，由前端控制）

### 8.3 权限控制

| 角色       | 创建 | 编辑 | 删除 | 测试连接 | 查看列表 | 查看详情 | 预览数据 |
| ---------- | ---- | ---- | ---- | -------- | -------- | -------- | -------- |
| 超管       | ✅   | ✅   | ✅   | ✅       | ✅       | ✅       | ✅       |
| 租户管理员 | ✅   | ✅   | ✅   | ✅       | ✅       | ✅       | ✅       |
| 报表设计者 | ✅   | ✅   | ❌   | ✅       | ✅       | ✅       | ✅       |

## 10. 上线方案

### 10.1 上线步骤

1. 执行 DDL 创建 `data_source_connection_log` 表
2. 部署包含数据源管理模块的 API 服务
3. 验证：创建 PostgreSQL 数据源 → 测试连接 → 查看表列表 → 预览数据
4. 对已有 `data_source` 表记录执行密码加密迁移脚本

### 10.2 灰度策略

数据源管理属于基础模块，无业务数据依赖，可全量上线。

### 10.3 监控告警

- 连接测试失败率 > 10% 触发告警
- 密码解密失败（密钥异常）触发 P1 告警

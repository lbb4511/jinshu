## 1. 背景与目标

### 1.1 问题背景

系统存在大量需要动态管理的配置项和业务枚举值：

| 问题                       | 影响                                   |
| -------------------------- | -------------------------------------- |
| 配置写在 `application.yml` | 修改需重启服务，运维成本高             |
| 业务枚举值无统一管理       | 各模块自行硬编码，前端重复维护下拉选项 |
| 开关型配置无热更新         | 水印、脱敏等功能无法线上动态启停       |

### 1.2 设计目标

| 指标     | 要求                                      |
| -------- | ----------------------------------------- |
| 热更新   | 系统配置修改后 1 分钟内生效（缓存刷新）   |
| 分层管理 | 系统级 > 租户级 > 用户级，优先级逐级覆盖  |
| 审计覆盖 | 所有配置修改操作 100% 记录审计日志        |
| 字典查询 | 字典 API 响应时间 < 50ms（Caffeine 缓存） |

### 1.3 非目标

- 不实现配置版本管理和回滚（二期）
- 不实现用户级个性化配置（当前最高到租户级）

## 2. 需求分析

### 2.1 功能需求

- [x] **系统配置管理**：新增/编辑/删除配置项，按分组展示
- [x] **配置分层**：系统级（全局默认）→ 租户级（覆盖默认）→ 用户级（预留）
- [x] **配置热更新**：修改后通过 Redis Pub/Sub 通知所有实例刷新
- [x] **字典管理**：字典组（sys_dict）与字典项（sys_dict_item）的 CRUD
- [x] **字典查询 API**：前端通过 `GET /api/dict/{dictCode}` 获取下拉选项
- [x] **开关类配置**：水印开关、脱敏开关、注册开关等 boolean 型配置
- [x] **审计记录**：所有配置/字典修改 100% 记录

### 2.2 用户故事

```
作为 系统管理员，我希望 动态调整文件上传大小限制，
以便 无需重启服务即可应对不同场景需求。

作为 前端开发者，我希望 通过一个统一接口获取所有下拉选项，
以便 无需在前端硬编码枚举值。

作为 安全管理员，我希望 紧急情况下能一键关闭所有导出功能，
以便 在发现安全漏洞时快速止血。
```

### 2.3 边界场景

| 场景                   | 处理方式                                     |
| ---------------------- | -------------------------------------------- |
| 系统级配置被租户级覆盖 | 查询时逐级合并：先查租户级，无则取系统级     |
| 配置值类型不匹配       | 配置表存储 `value_type` 字段，读取时校验转换 |
| 删除被引用的字典项     | 字典项 `is_enabled=false`，不物理删除        |
| 缓存刷新传播延迟       | Redis Pub/Sub 通知，最大延迟 < 5s            |
| 配置项 key 冲突        | 数据库 UNIQUE(tenant_id, config_key) 约束    |
| 租户级覆盖被删除的配置 | 租户级配置删除后自动回退到系统级             |

## 3. 架构设计

### 3.1 整体流程

```
管理员
  │
  ▼
ConfigController / DictController  ── @AuditLog
  │
  ▼
SysConfigService / DictService  ── 缓存更新、租户隔离
  │
  ├──► SysConfigMapper / DictMapper  ── 数据库 CRUD
  ├──► ConfigCacheManager  ── 多级缓存（Caffeine + Redis）
  └──► Redis Pub/Sub  ── 配置变更通知
```

### 3.2 配置分层模型

```
查询优先级（由高到低）：
  用户级配置（预留）
    └─ 未设置 → 租户级配置
      └─ 未设置 → 系统级配置（tenant_id = 0）
        └─ 未设置 → application.yml 默认值（硬编码兜底）
```

### 3.3 模块划分

| 模块                | 职责                         | 所属模块 |
| ------------------- | ---------------------------- | -------- |
| SysConfigController | 配置 CRUD API                | `api/`   |
| DictController      | 字典查询 API（用户侧）       | `api/`   |
| DictAdminController | 字典管理 API（管理侧）       | `api/`   |
| SysConfigService    | 配置业务逻辑、分层合并、缓存 | `api/`   |
| DictService         | 字典业务逻辑、缓存           | `api/`   |
| ConfigCacheManager  | 多级缓存管理、Pub/Sub 通知   | `api/`   |

## 4. 数据库设计

### 4.1 sys_config 表

| 字段           | 类型         | 说明                                                       |
| -------------- | ------------ | ---------------------------------------------------------- |
| `id`           | BIGSERIAL PK | 主键                                                       |
| `tenant_id`    | BIGINT       | 租户 ID（0 = 系统级默认配置）                              |
| `config_key`   | VARCHAR(200) | 配置键，如 `file.max_upload_size_mb`                       |
| `config_value` | TEXT         | 配置值（统一存为文本）                                     |
| `value_type`   | VARCHAR(20)  | STRING / INTEGER / BOOLEAN / JSON                          |
| `config_group` | VARCHAR(100) | 配置分组：FILE / SECURITY / IMPORT / EXPORT / PDF / SYSTEM |
| `description`  | TEXT         | 配置说明                                                   |
| `is_encrypted` | BOOLEAN      | 是否加密存储（敏感配置如密钥）                             |
| `created_by`   | BIGINT       | 创建人                                                     |
| `updated_by`   | BIGINT       | 修改人                                                     |
| `created_at`   | TIMESTAMP    | 创建时间                                                   |
| `updated_at`   | TIMESTAMP    | 更新时间                                                   |

**预置系统级配置项**：

| config_key                         | value_type | config_group | 默认值               | 说明             |
| ---------------------------------- | ---------- | ------------ | -------------------- | ---------------- |
| `file.max_upload_size_mb`          | INTEGER    | FILE         | 100                  | 最大上传文件(MB) |
| `file.allowed_import_types`        | JSON       | FILE         | ["xlsx","xls","csv"] | 允许的导入格式   |
| `import.batch_size`                | INTEGER    | IMPORT       | 1000                 | 导入批次大小     |
| `export.max_rows`                  | INTEGER    | EXPORT       | 100000               | 单次导出行数上限 |
| `security.login_fail_lock_count`   | INTEGER    | SECURITY     | 5                    | 登录失败锁定阈值 |
| `security.login_fail_lock_minutes` | INTEGER    | SECURITY     | 30                   | 锁定分钟数       |
| `security.password_min_length`     | INTEGER    | SECURITY     | 8                    | 密码最小长度     |
| `security.audit_retention_months`  | INTEGER    | SECURITY     | 6                    | 审计日志保留月数 |
| `pdf.render_timeout_seconds`       | INTEGER    | PDF          | 60                   | PDF 渲染超时(秒) |
| `pdf.max_concurrent_renders`       | INTEGER    | PDF          | 5                    | 最大并发渲染数   |
| `feature.watermark_enabled`        | BOOLEAN    | SYSTEM       | true                 | 水印功能开关     |
| `feature.desensitize_enabled`      | BOOLEAN    | SYSTEM       | true                 | 数据脱敏开关     |
| `feature.allow_registration`       | BOOLEAN    | SYSTEM       | false                | 开放注册开关     |

## 5. API 接口设计

### 5.1 接口列表

| 方法                   | 路径                                            | 说明               | 权限            |
| ---------------------- | ----------------------------------------------- | ------------------ | --------------- |
| **系统配置管理**       |                                                 |                    |                 |
| GET                    | `/api/admin/configs`                            | 配置列表           | 超管+租户管理员 |
| GET                    | `/api/admin/configs/{key}`                      | 单个配置详情       | 超管+租户管理员 |
| PUT                    | `/api/admin/configs/{key}`                      | 修改配置           | 超管+租户管理员 |
| DELETE                 | `/api/admin/configs/{key}`                      | 删除配置（租户级） | 超管+租户管理员 |
| **字典管理**           |                                                 |                    |                 |
| POST                   | `/api/admin/dicts`                              | 创建字典组         | 超管+租户管理员 |
| PUT                    | `/api/admin/dicts/{dictCode}`                   | 编辑字典组         | 超管+租户管理员 |
| DELETE                 | `/api/admin/dicts/{dictCode}`                   | 删除字典组         | 超管            |
| POST                   | `/api/admin/dicts/{dictCode}/items`             | 新增字典项         | 超管+租户管理员 |
| PUT                    | `/api/admin/dicts/{dictCode}/items/{id}`        | 编辑字典项         | 超管+租户管理员 |
| PATCH                  | `/api/admin/dicts/{dictCode}/items/{id}/status` | 启停字典项         | 超管+租户管理员 |
| **字典查询（用户侧）** |                                                 |                    |                 |
| GET                    | `/api/dict/{dictCode}`                          | 获取字典项列表     | 登录用户        |

### 5.2 请求/响应示例

**获取配置（自动分层合并）**：

```json
// GET /api/admin/configs/file.max_upload_size_mb
// 租户 1001 设有自定义值时返回租户级，否则返回系统级
{
  "code": 0,
  "data": {
    "configKey": "file.max_upload_size_mb",
    "configValue": "200",
    "valueType": "INTEGER",
    "configGroup": "FILE",
    "source": "TENANT",
    "systemDefault": "100"
  }
}
```

**修改租户级配置**：

```json
// PUT /api/admin/configs/file.max_upload_size_mb
// 请求体（由租户管理员调用，自动设置 tenant_id）
{ "configValue": "200" }

// 响应
{
  "code": 0,
  "message": "配置已更新，预计 1 分钟内生效"
}
```

**获取字典（前端下拉选项）**：

```json
// GET /api/dict/REPORT_TYPE
{
  "code": 0,
  "data": [
    { "value": "FINANCIAL", "label": "财务报表", "sortOrder": 1 },
    { "value": "OPERATION", "label": "运营报表", "sortOrder": 2 },
    { "value": "HR", "label": "人力报表", "sortOrder": 3 }
  ]
}
```

## 6. 异常处理

| 场景                 | 错误码 | HTTP 状态码 | 处理方式                           |
| -------------------- | ------ | ----------- | ---------------------------------- |
| 配置类型转换失败     | 40001  | 400         | 返回"配置值类型不匹配"             |
| 配置项不存在         | 40400  | 404         | 返回"配置项不存在"                 |
| 系统级配置不可删除   | 40300  | 403         | 仅超管可删除系统级配置             |
| 字典编码重复         | 40900  | 409         | 返回"字典编码已存在"               |
| 字典被引用不可删除   | 40901  | 409         | 返回引用该字典的字段列表           |
| 非超管操作系统级配置 | 40300  | 403         | 返回"仅超级管理员可操作系统级配置" |

## 7. 性能考量

| 考量点       | 策略                                             |
| ------------ | ------------------------------------------------ |
| 配置查询     | Caffeine 本地缓存（TTL 10 分钟）+ Redis 二级缓存 |
| 字典查询     | Caffeine 本地缓存（TTL 1 小时），变更时通知失效  |
| 缓存预热     | 应用启动时预加载全部系统级配置和字典             |
| 配置变更传播 | Redis Pub/Sub 通知所有实例，最大延迟 < 5s        |
| 字典项数量大 | 单个字典组建议 ≤ 500 项，超过建议分拆            |

**缓存 Key 设计**：

```
config:{tenant_id}:{config_key}  → 配置值
dict:{tenant_id}:{dict_code}     → 字典项列表
```

## 8. 安全考量

| 考量点       | 策略                                             |
| ------------ | ------------------------------------------------ |
| 敏感配置加密 | `is_encrypted=true` 的配置使用 AES-256-GCM 加密  |
| 租户隔离     | 租户只能修改自己的配置，看不到其他租户配置       |
| 系统级保护   | 系统级配置仅超管可修改，租户管理员可查看但不可改 |
| 审计覆盖     | L3 级别事件，记录变更前后配置值                  |
| 变更通知     | 配置修改后通知所有实例刷新，但不暴露变更内容     |

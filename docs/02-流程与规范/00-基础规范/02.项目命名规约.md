## 1. 核心原则

| 原则           | 说明                         |
| -------------- | ---------------------------- |
| **可读性优先** | 宁愿长一点，也要让人一看就懂 |
| **英文命名**   | 禁止拼音，更禁止拼音缩写     |
| **风格统一**   | 同一类事物用同样的命名模式   |
| **见名知意**   | 不需要注释就能理解用途       |

## 2. Java 代码命名

### 2.1 包名（Package）

| 类型   | 规则                             | 示例                          |
| ------ | -------------------------------- | ----------------------------- |
| 全小写 | 全部小写，无下划线、无大写       | `com.jinshu`                  |
| 分层包 | controller, service, dao, entity | `com.jinshu.controller`       |
| 公共包 | common 下按功能分                | `com.jinshu.common.exception` |
| 配置包 | 全部放在 config 下               | `com.jinshu.config`           |

**目录结构标准：**

```
com.jinshu
├── controller/          # 控制层
├── service/             # 业务层
│   └── impl/            # Service 实现类
├── dao/                 # 数据访问层（Mapper）
├── entity/              # 数据实体（DO）
├── dto/                 # 数据传输对象
│   ├── request/         # 请求 DTO
│   └── response/        # 响应 DTO
├── vo/                  # 视图对象（返回给前端）
├── common/              # 公共组件
│   ├── exception/       # 异常类
│   ├── result/          # 统一返回
│   ├── context/         # 上下文
│   ├── annotation/      # 自定义注解
│   └── utils/           # 工具类
├── config/              # 配置类
├── security/            # 安全组件（AOP、拦截器）
└── batch/               # 批处理
    ├── reader/
    ├── processor/
    └── writer/
```

### 2.2 类名（Class）

| 类类型        | 规则                        | 示例                       |
| ------------- | --------------------------- | -------------------------- |
| 普通类        | 大驼峰（PascalCase）        | `ReportService`            |
| 接口          | 形容词或名词（不加 I 前缀） | `ReportService`            |
| 接口实现      | 接口名 + Impl               | `ReportServiceImpl`        |
| Controller    | 业务名 + Controller         | `ReportController`         |
| Service       | 业务名 + Service            | `ReportService`            |
| DAO/Mapper    | 业务名 + Mapper             | `ReportMapper`             |
| 实体类        | 业务名                      | `Report`, `Tenant`, `User` |
| DTO 请求      | 业务名 + 操作 + Request     | `ReportCreateRequest`      |
| DTO 响应      | 业务名 + 操作 + Response    | `ReportQueryResponse`      |
| VO            | 业务名 + VO                 | `ReportVO`                 |
| 异常类        | 业务名 + Exception          | `BusinessException`        |
| 工具类        | 功能名 + Utils              | `DateUtils`, `HashUtils`   |
| 拦截器/过滤器 | 功能名 + Interceptor/Filter | `TenantInterceptor`        |
| 切面类        | 功能名 + Aspect             | `AuditLogAspect`           |
| 配置类        | 功能名 + Config             | `MyBatisConfig`            |
| 枚举类        | 功能名 + Enum               | `ErrorCode`, `TaskStatus`  |

❌ **错误示例**：

- `IReportService`（不要加 I 前缀）
- `ReportServiceI`（不要加 I 后缀）
- `ReportDto`（DTO 要大写：`ReportDTO`）
- `reportService`（类名首字母要大写）

### 2.3 方法名（Method）

| 操作类型  | 前缀            | 示例                              |
| --------- | --------------- | --------------------------------- |
| 查询单个  | get             | `getById`, `getReport`            |
| 查询多个  | list/query      | `listByTenantId`, `queryReport`   |
| 新增      | create/save/add | `createReport`, `saveReport`      |
| 修改      | update/edit     | `updateReport`, `editStatus`      |
| 删除      | delete/remove   | `deleteById`, `removeReport`      |
| 批量操作  | batch           | `batchDelete`, `batchInsert`      |
| 导入      | import          | `importExcel`                     |
| 导出      | export          | `exportExcel`, `exportPdf`        |
| 校验      | validate/check  | `validateData`, `checkPermission` |
| 转换      | convert/parse   | `convertToVO`, `parseJson`        |
| 处理/执行 | process/execute | `processTask`, `executeJob`       |

**布尔值方法名：**

```java
// ✅ 好的命名
boolean hasPermission();
boolean isVip();
boolean exists();
boolean isDeleted();

// ❌ 不要这样
boolean getHasPermission();
boolean checkIsVip();
```

---

### 2.4 变量名（Variable）

| 类型     | 规则                | 示例                             |
| -------- | ------------------- | -------------------------------- |
| 普通变量 | 小驼峰（camelCase） | `reportId`, `tenantName`         |
| 常量     | 全大写 + 下划线     | `MAX_SIZE`, `DEFAULT_PAGE_SIZE`  |
| 集合变量 | 类型后缀            | `reportList`, `userMap`, `idSet` |
| 泛型     | 单个大写字母        | `T`, `E`, `K`, `V`               |

**常见缩写约定：**

| 全称           | 缩写         | 说明                    |
| -------------- | ------------ | ----------------------- |
| Identification | id           | 统一用 id，不要用 ID/Id |
| Configuration  | config       | 配置                    |
| Parameter      | param        | 参数                    |
| Database       | db           | 数据库                  |
| Message        | msg          | 消息                    |
| Number         | num          | 数量                    |
| Result         | res / result | 结果                    |
| Exception      | e            | 异常对象                |

❌ **禁止的命名：**

```java
// ❌ 不要用单个字母（循环除外）
User a;
Report b;

// ❌ 不要用拼音
User yongHu;
Report baoBiao;

// ❌ 不要用无意义的数字
Report report2;
User user3;
```

---

### 2.5 常量定义

```java
// ✅ 推荐：按功能分组，用接口或类组织
public interface ReportConstants {
    int MAX_ROWS = 1000000;
    int BATCH_SIZE = 1000;
    String EXPORT_TYPE_EXCEL = "excel";
}

// ❌ 不要散落在各个类里
```

## 3. 数据库命名

### 3.1 表名

| 规则            | 说明                       | 示例                         |
| --------------- | -------------------------- | ---------------------------- |
| 全小写 + 下划线 | 统一小写，单词间下划线     | `report`, `report_data`      |
| 模块前缀        | 同一模块的表用相同前缀     | `task_xxx`, `datasource_xxx` |
| 多租户表        | 所有业务表必须有 tenant_id |                              |

**标准表名：**

```sql
-- 用户与租户
tenant               -- 租户表
users                -- 用户表（user 是关键字，加 s）

-- 报表模块
report_metadata      -- 报表元数据
report_data          -- 报表数据宽表
report_workflows     -- 工作流

-- 任务模块
task                 -- 任务表
task_log             -- 任务日志

-- 审计与日志
audit_log            -- 审计日志
audit_root_hash      -- 审计根哈希

-- 系统
sys_config           -- 系统配置
sys_dict             -- 字典表
```

---

### 3.2 字段名

| 规则            | 说明               | 示例                                     |
| --------------- | ------------------ | ---------------------------------------- |
| 全小写 + 下划线 |                    | `tenant_id`, `created_at`                |
| 主键            | id（不用表名前缀） | `id`                                     |
| 外键            | 表名\_id           | `tenant_id`, `report_id`                 |
| 时间字段        | xxx_at             | `created_at`, `updated_at`, `deleted_at` |
| 布尔字段        | is_xxx             | `is_deleted`, `is_vip`                   |
| 状态字段        | status             | 用字符串或数字                           |
| JSON 字段       | xxx_json           | `data_json`, `extra_json`                |

**通用字段约定（所有业务表必含）：**

```sql
id              BIGSERIAL PRIMARY KEY
tenant_id       BIGINT NOT NULL
created_at      TIMESTAMP DEFAULT NOW()
created_by      BIGINT
updated_at      TIMESTAMP DEFAULT NOW()
updated_by      BIGINT
is_deleted      BOOLEAN DEFAULT FALSE
```

## 5. 文档命名

### 5.1 设计文档命名

```
docs/
├── 01-立项与架构/
│   ├── 01.需求文档.md
│   ├── 02.概要设计.md
│   ├── 03.技术选型决策记录.md
│   └── 04.安全合规架构设计.md
├── 02-模块设计/
│   ├── 01.报表导入模块设计.md
│   ├── 02.报表导出模块设计.md
│   └── 03.权限与多租户模块设计.md
├── 03-功能设计/
│   ├── 01.报表预览水印功能设计.md
│   └── 02.大任务自动分片功能设计.md
└── 04-运维部署/
    └── 01.K3s部署指南.md
```

**命名规则：**

- 两位数字前缀，保证排序正确
- 中文命名，清晰说明内容
- 后缀固定：`xxx设计.md` / `xxx指南.md` / `xxx手册.md`

### 5.2 API 接口命名

```
GET    /api/reports              # 查询列表（复数）
GET    /api/reports/{id}         # 查询单个
POST   /api/reports              # 新增
PUT    /api/reports/{id}         # 修改
DELETE /api/reports/{id}         # 删除
```

**URL 规则：**

- 全小写
- 用名词复数形式
- 单词间用减号 `-`
- 层级不超过 3 层

```http
// ✅ 好的
GET /api/task-progress

// ❌ 不好的
GET /api/taskProgress          // 驼峰不行
GET /api/task_progress         // 下划线不行
GET /api/getTaskProgress       // 不要加动词
```

## 6. Git 命名规范

### 6.1 分支名

```
main                  # 主分支，生产环境
develop               # 开发分支

feature/xxx           # 功能分支
feature/report-import

bugfix/xxx            # Bug 修复分支
bugfix/export-error

hotfix/xxx            # 紧急修复
hotfix/prod-oom
```

### 6.2 Commit Message

**格式：**

```
<type>(<scope>): <subject>

<body>
```

**type 类型：**

| type     | 说明               |
| -------- | ------------------ |
| feat     | 新功能             |
| fix      | 修复 bug           |
| docs     | 文档更新           |
| style    | 代码格式调整       |
| refactor | 重构               |
| test     | 测试相关           |
| chore    | 构建/工具/依赖更新 |

**示例：**

```
feat(report): 增加 Excel 导入功能

- 实现 EasyExcel 流式读取
- 增加数据校验
- 增加批量写入优化

Closes #123
```

## 7. 反模式速查

| ❌ 不要                    | ✅ 应该                          |
| -------------------------- | -------------------------------- |
| `rpt`                      | `report`（不要瞎缩写）           |
| `getAllData()`             | `listReport()`（清晰说明查什么） |
| `handleData()`             | `processReportData()`（太模糊）  |
| `flag`, `status`           | `isVip`, `auditStatus`（更具体） |
| `temp`, `tmp`              | 尽量说明是什么临时变量           |
| `User user1`, `User user2` | 想办法区分：`newUser`, `dbUser`  |
| `abc123`, `test`           | 开发测试代码不要提交             |

## 8. 优先级说明

| 优先级 | 场景         | 说明                               |
| ------ | ------------ | ---------------------------------- |
| 🔴 P0  | 核心领域概念 | 必须严格统一，比如报表、租户、任务 |
| 🟡 P1  | 通用命名     | 尽量遵守，比如 get/list/create     |
| 🟢 P2  | 临时变量     | 合理即可，循环变量可以用 i/j/k     |

> 最关键的是：**同一概念在全项目中用同一个词！**
>
> 不要有的地方叫 `报表`，有的地方叫 `报告`
> 不要有的地方叫 `租户`，有的地方叫 `企业`
> 不要有的地方叫 `任务`，有的地方叫 `作业`

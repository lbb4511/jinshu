## 1. 开源协议

本项目采用 **Apache License 2.0**，这是一个宽松、无需额外授权的开源许可协议，特别适合企业级项目。

### Apache 2.0 协议核心要点

- ✅ 无需授权即可使用
- ✅ 允许商业使用
- ✅ 允许修改和分发
- ✅ 明确授予专利权保护
- ✅ 起诉专利侵权则自动终止许可
- ⚠️ 必须包含原始版权、许可声明和 NOTICE 文件
- ⚠️ 修改文件必须声明已修改

### 为什么选择 Apache 2.0

相比 MIT 协议，Apache 2.0 增加了明确的专利保护条款，对企业用户更加友好，是目前企业级开源项目最广泛采用的协议之一。

### ❌ 明确禁止使用的商业技术

- 不得引入任何需要购买 License 的商业源代码
- 不得使用专有闭源的 SDK 或组件库
- 不得调用付费的 SaaS API 作为核心功能依赖
- 不得引入包含商业授权限制的第三方依赖

## 3. 禁止付费商业代码和软件

为确保项目的完全开源和自由使用，本项目严格遵守以下规定：

### 3.1 禁止引入的内容

- ❌ **禁止引入付费商业源代码**：不得将任何需要付费购买的商业软件代码引入本项目
- ❌ **禁止引入专有闭源组件**：不得使用闭源的商业库、框架或组件
- ❌ **禁止引入有商业授权限制的依赖**：所有第三方依赖必须采用兼容的开源协议
- ❌ **禁止添加付费功能限制**：不得在代码中添加需要付费解锁的功能限制
- ❌ **禁止调用核心付费 SaaS 服务**：核心功能不得依赖付费 API

### 3.2 允许的内容

- ✅ **纯开源代码**：所有代码必须是完全开源的
- ✅ **Apache/MIT/GPL 等开源协议的依赖**：只可使用公认开源协议的第三方库
- ✅ **自由贡献**：欢迎任何形式的开源贡献

## 4. 第三方依赖审核流程

所有引入的第三方依赖必须符合以下要求：

1. 必须有明确的开源协议声明（在 `LICENSE` 或 `package.json/pom.xml` 中声明）
2. 协议必须与 Apache 2.0 协议兼容
3. 不得包含任何商业授权要求
4. 新依赖引入必须提交 PR 并注明开源协议类型

## 5. 贡献者须知

所有向本项目贡献代码的开发者均视为同意：

- 其贡献的代码采用 Apache 2.0 协议
- 贡献内容不包含任何付费商业代码
- 不引入任何有商业授权限制的依赖
- 修改的文件必须声明已修改

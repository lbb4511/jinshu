## Git 工作流

### 分支策略

使用 **Git Flow 简化版**：

| 分支 | 说明 | 生命周期 |
|------|-----|---------|
| `main` | 稳定生产版本 | 永久 |
| `develop` | 开发集成分支 | 永久 |
| `feature/*` | 功能开发分支 | 临时，合并后删除 |
| `hotfix/*` | 紧急修复分支 | 临时，合并后删除 |
| `release/*` | 发布预发布分支 | 临时，发布后删除 |

### 提交信息规范

使用 **约定式提交 (Conventional Commits)**：

```
<type>(<scope>): <subject>

<body>
```

**Type 类型**：

| 类型 | 说明 |
|------|-----|
| `feat` | 新功能 |
| `fix` | 缺陷修复 |
| `docs` | 文档更新 |
| `style` | 代码格式（不影响功能） |
| `refactor` | 重构 |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具/依赖 |

**示例**：

```
feat(report): 新增 PDF 导出功能

- 支持分页打印 CSS 控制
- 新增进度回调 Redis 键值设计
```

## 语义化版本规范

### 版本号格式

```
MAJOR.MINOR.PATCH[-PRERELEASE][+BUILD]
```

| 部分 | 说明 | 更新时机 |
|------|------|---------|
| MAJOR | 主版本 | 不兼容的 API 变更 |
| MINOR | 次版本 | 向下兼容的功能新增 |
| PATCH | 补丁 | 向下兼容的问题修复 |

### 示例

| 版本号 | 说明 |
|------|------|
| `1.0.0` | 首个稳定版本 |
| `1.1.0` | 新增功能 |
| `1.1.1` | Bug 修复 |
| `2.0.0` | 破坏性变更 |
| `1.2.0-beta.1` | Beta 预发布版本 |

### 版本升级策略

| 阶段 | 版本规则 | 示例 |
|------|--------|-----|
| **开发期 (0.x.x) | 快速迭代，MINOR 作为主版本 | `0.1.0` → `0.2.0` |
| **稳定期** (1.x.x+) | 严格遵循语义化版本 | `1.0.0` → `1.1.0` → `1.1.1` → `2.0.0` |

## 依赖版本管理

### 依赖范围符号

| 符号 | 说明 | 示例 |
|------|------|------|
| `^` | 兼容更新（推荐） | `^1.2.3` → 1.x.x |
| `~` | 补丁更新 | `~1.2.3` → 1.2.x |
| `=` | 固定版本 | `=1.2.3` |

### Gradle 依赖升级原则

| 依赖类型 | 版本策略 |
|----------|---------|
| **Spring Boot / 核心框架** | 跟随 Spring Boot BOM 统一管理 |
| **MyBatis / 数据库驱动** | 固定到具体版本，季度检查更新 |
| **EasyExcel / PDFBox** | 固定具体版本，半年评估一次 |
| **工具库 (Lombok / Commons)** | 跟随 Spring Boot BOM 或固定 |

### 依赖升级安全检查清单

升级依赖前检查：

- [ ] 查阅 CHANGELOG 确认无破坏性变更
- [ ] 检查新依赖的开源协议兼容性
- [ ] 检查是否存在已知安全漏洞 (CVE)
- [ ] 运行完整测试集通过
- [ ] 更新对应模块集成测试通过
- [ ] 性能无明显性能回退（超过 10%

### 依赖锁定

使用 **Gradle 版本目录 `libs.versions.toml`** 管理后端依赖：

```toml
[versions]
spring-boot = "4.0.6"
mybatis = "3.0.5"

[libraries]
spring-boot-starter-web = { module = "org.springframework.boot:spring-boot-starter-web", version.ref = "spring-boot" }
spring-boot-starter-aspectj = { module = "org.springframework.boot:spring-boot-starter-aspectj", version.ref = "spring-boot" }
```

前端依赖通过 **`package.json`** 锁定：

```json
{
  "dependencies": {
    "react": "^19.2.6",
    "react-dom": "^19.2.6",
    "react-router-dom": "^7.15.0",
    "primereact": "^10.9.7",
    "echarts": "^6.0.0",
    "axios": "^1.8.4",
    "dayjs": "^1.11.19"
  },
  "devDependencies": {
    "vite": "^8.0.12",
    "typescript": "^6.0.3",
    "sass": "^1.86.0"
  }
}
```

## 发布流程

### 发布前检查清单

- [ ] 所有功能已合并到 `develop`
- [ ] 单元测试通过率 100%
- [ ] 集成测试通过率 100%
- [ ] Code Review 完成
- [ ] 数据库迁移脚本已编写
- [ ] 文档已更新
- [ ] 回滚方案已准备

### 发布步骤

1. **创建发布分支
   ```bash
   git checkout -b release/1.1.0
   ```

2. **更新版本号**
   - 更新 `build.gradle.kts` / `package.json`
   - 更新 CHANGELOG
   - 提交: `chore: bump version to 1.1.0`

3. **测试验证**
   - 运行完整测试
   - 做发布前预发布验证

4. **合并发布**
   ```bash
   git checkout main
   git merge --no-ff release/1.1.0
   git tag -a v1.1.0 -m "Release version 1.1.0"
   git push origin main --tags

   git checkout develop
   git merge --no-ff release/1.1.0
   ```

5. **删除发布分支**
   ```bash
   git branch -d release/1.1.0
   ```

6. **部署发布**
   - 按部署流程操作

### 紧急修复 (Hotfix)

```bash
git checkout -b hotfix/1.0.1 main
# 修复问题
git commit -m "fix: xxx 修复 xxx 问题"
git tag -a v1.0.1 -m "Hotfix version 1.0.1"
git checkout main
git merge --no-ff hotfix/1.0.1
git checkout develop
git merge --no-ff hotfix/1.0.1
git branch -d hotfix/1.0.1
```

## 数据库迁移

### 迁移脚本命名

```
V{VERSION}__{DESCRIPTION}.sql
```

**示例**：
```
V1__init_schema.sql
V2__add_tenant_column.sql
V3__add_audit_log_table.sql
```

### Flyway 迁移原则

| 原则 | 说明 |
|------|------|
| ✅ 迁移脚本只读 | 提交后不可修改！ |
| ✅ 向下兼容 | 迁移必须支持回滚（尽可能） |
| ✅ 小步快跑 | 单次迁移不超过 1 分钟 |
| ❌ 禁止 `DROP TABLE` | 除非确定数据已归档 |
| ❌ 禁止大表 `ALTER TABLE` | 大表结构变更需分阶段处理 |

### 大表变更策略

对于数据量超过 100 万的表：

1. **新增字段**
   - 先加可空字段，后台异步回填数据

2. **删除字段**
   - 先让代码不再访问，观察 2 周后再删

3. **变更字段**
   - 先建新字段，同步双写，再切流量

## CHANGELOG 规范

每个发布必须更新 `CHANGELOG.md`：

```markdown
## [1.1.0] - 2026-05-20
### 新增
- 报表 PDF 导出功能

### 改进
- 优化导入性能 30%

### 修复
- 修复租户隔离在批量查询问题
```

格式参考：[Keep a Changelog](https://keepachangelog.com/)

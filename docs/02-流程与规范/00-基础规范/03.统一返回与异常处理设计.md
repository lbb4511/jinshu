## 1. 设计目标

1. **前后端契约标准化**：所有 API 返回格式统一
2. **异常集中化**：业务代码不手动 try-catch 包装 Result
3. **错误码可枚举**：所有错误场景有明确编码，便于排查
4. **日志可追溯**：异常发生时自动记录上下文信息

## 2. 统一返回格式设计

### 2.1 Result<T> 结构

```json
{
  "code": 0,
  "message": "success",
  "data": {
    // 业务数据
  }
}
```

| 字段    | 类型   | 说明                   |
| ------- | ------ | ---------------------- |
| code    | int    | 0=成功，其他=错误码    |
| message | string | 提示信息               |
| data    | T      | 泛型，返回具体业务数据 |

### 2.2 PageResult<T> 分页结构

```json
{
  "list": [],
  "total": 100,
  "pageNum": 1,
  "pageSize": 20
}
```

## 3. 错误码设计

### 3.1 错误码分段

| 码段      | 错误类型      | 示例                                 |
| --------- | ------------- | ------------------------------------ |
| 0         | 成功          | 0                                    |
| 400-499   | 客户端错误    | 400=参数错误，401=未认证，403=无权限 |
| 500-599   | 服务端错误    | 500=系统内部错误                     |
| 1000-1999 | 租户/用户模块 | 1001=租户不存在                      |
| 2000-2999 | 报表模块      | 2001=报表不存在                      |
| 3000-3999 | 任务模块      | 3001=任务不存在                      |
| 4000-4999 | 文件模块      | 4001=文件类型不支持                  |
| 5000-5999 | 数据源模块    | 5001=数据源连接失败                  |

### 3.2 ErrorCode 枚举设计

```java
public enum ErrorCode {
    SUCCESS(0, "成功"),
    PARAM_ERROR(400, "参数错误"),
    UNAUTHORIZED(401, "未授权"),
    FORBIDDEN(403, "无权限"),

    TENANT_NOT_FOUND(1001, "租户不存在"),
    REPORT_NOT_FOUND(2001, "报表不存在"),
    // ...
}
```

## 4. 异常体系设计

### 4.1 异常类层次

```
Throwable
└── Exception
    └── RuntimeException
        └── BusinessException    # 业务异常，本项目唯一受检异常
```

### 4.2 异常抛出规范

**✅ 推荐写法**：

```java
if (tenant == null) {
    throw new BusinessException(ErrorCode.TENANT_NOT_FOUND);
}
```

**❌ 禁止写法**：

```java
// 禁止：业务代码手动包装 Result 返回
return Result.error(1001, "租户不存在");
```

### 4.3 全局异常拦截流程

```
业务代码抛出 BusinessException
        ↓
GlobalExceptionHandler 捕获
        ↓
记录 ERROR 级别日志 (含上下文)
        ↓
包装为 Result.error(code, message) 返回
        ↓
前端收到标准格式响应
```

## 5. 异常类型说明

| 异常类型            | 场景示例               | HTTP 状态码 | 日志级别 |
| ------------------- | ---------------------- | ----------- | -------- |
| BusinessException   | 租户不存在、报表已发布 | 200         | WARN     |
| ValidationException | 参数校验失败           | 400         | INFO     |
| Exception           | NPE、数组越界等        | 500         | ERROR    |

## 6. Controller 层编码规范

### 6.1 统一返回类型

**✅ 所有 Controller 方法必须返回 Result<T>**

```java
// ✅ 正确
@GetMapping("/{id}")
public Result<Report> getById(@PathVariable Long id) {
    Report report = reportService.getById(id);
    return Result.success(report);
}

// ❌ 禁止：直接返回业务对象
@GetMapping("/{id}")
public Report getById(@PathVariable Long id) {
    return reportService.getById(id);
}
```

### 6.2 Service 层不感知 Result

Service 层只负责抛出 BusinessException，不包装 Result

## 7. 前端类型对应

```typescript
// services/request.ts
export interface Result<T = any> {
  code: number;
  message: string;
  data: T;
}

export interface PageResult<T> {

  total: number;
  pageNum: number;
  pageSize: number;
}
```

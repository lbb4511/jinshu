## 1. 合规要求

| 等保三级要求      | 实现方案                     |
| ----------------- | ---------------------------- |
| 日志保存 ≥ 6 个月 | 数据库分区按月归档           |
| 日志不可篡改      | 数据库触发器 + Merkle 哈希树 |
| 日志不可删除      | REVOKE UPDATE/DELETE 权限    |
| 操作全记录        | 所有写操作自动记录           |

## 2. 审计字段设计

### 2.1 audit_log 表结构

```sql
CREATE TABLE audit_log (
    id              BIGSERIAL PRIMARY KEY,
    tenant_id       BIGINT NOT NULL,
    user_id         BIGINT NOT NULL,
    username        VARCHAR(64) NOT NULL,
    operation       VARCHAR(32) NOT NULL,    -- CREATE/UPDATE/DELETE/EXPORT
    target_type     VARCHAR(32),             -- REPORT/DATASOURCE/TASK
    target_id       BIGINT,
    target_name     VARCHAR(255),
    ip_address      VARCHAR(45) NOT NULL,
    user_agent      VARCHAR(512),
    request_params  JSONB,
    response_data   JSONB,
    status          VARCHAR(16) NOT NULL,    -- SUCCESS/FAILED
    error_message   TEXT,
    created_at      TIMESTAMP DEFAULT NOW(),
    log_hash        VARCHAR(64) NOT NULL     -- 本条日志哈希
);

-- 审计日志防篡改触发器
CREATE OR REPLACE FUNCTION prevent_audit_modify()
RETURNS TRIGGER AS $$
BEGIN
    RAISE EXCEPTION '审计日志不可修改或删除';
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER audit_log_no_update
    BEFORE UPDATE OR DELETE ON audit_log
    FOR EACH ROW EXECUTE FUNCTION prevent_audit_modify();
```

## 3. 防篡改设计

### 3.1 Merkle 哈希树（每小时根哈希）

```
每小时生成根哈希存入 audit_root_hash 表：

    [00:00根哈希] ← 包含 00:00-00:59 所有日志
    [01:00根哈希] ← 包含 01:00-01:59 所有日志
    [02:00根哈希] ← ...

验证方式：
1. 取出某小时内所有日志
2. 重新计算根哈希
3. 与数据库中存储的根哈希比对
4. 不一致说明被篡改
```

### 3.2 哈希链计算

```
第 N 条日志的 hash = SHA256(第N-1条日志的 hash + 本条日志内容)
```

**篡改检测**：只要任意一条日志被修改，后续所有哈希都会断裂。

## 4. 采集方案

### 4.1 AOP 自动采集（90% 场景）

```java
@Aspect
@Component
public class AuditLogAspect {

    // 拦截所有 Post/Put/Delete 注解的方法
    @Around("@annotation(PostMapping) || ...")
    public Object around(ProceedingJoinPoint joinPoint) {
        // 1. 记录请求参数
        // 2. 执行方法
        // 3. 记录返回结果或异常
        // 4. 异步写入审计日志表
    }
}
```

## 5. 性能优化

| 优化点   | 方案                               |
| -------- | ---------------------------------- |
| 写入性能 | 异步批量写入（RabbitMQ 解耦）      |
| 查询性能 | 按 tenant_id + created_at 组合索引 |
| 存储成本 | 超过 6 个月的日志归档到对象存储    |

**写入性能指标**：峰值 TPS ≥ 1000

## 6. 三权分立要求

| 角色           | 权限                           |
| -------------- | ------------------------------ |
| **系统管理员** | 配置系统、管理用户             |
| **安全管理员** | 配置权限、审计策略             |
| **审计员**     | **仅查看**审计日志，无其他权限 |

**关键约束**：三种角色不可兼任，审计日志只有审计员可以查看。

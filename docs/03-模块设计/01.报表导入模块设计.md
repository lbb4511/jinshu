## 1. 设计目标

| 指标         | 要求                    |
| ------------ | ----------------------- |
| 最大支持行数 | 100 万行                |
| 内存占用峰值 | < 512MB (100万行导入时) |
| 导入速度     | > 2000 行/秒            |
| 失败重试     | 支持断点续传            |
| 进度可查     | 实时返回导入进度        |

---

## 2. 技术选型

| 技术点     | 选型方案                | 选型理由                              |
| ---------- | ----------------------- | ------------------------------------- |
| Excel 解析 | EasyExcel               | 基于 SAX 流式解析，内存友好，避免 OOM |
| 批量写入   | MyBatis BatchExecutor   | JDBC 批处理，减少网络往返             |
| 异步调度   | Spring Batch + RabbitMQ | 支持分片、重试、监控                  |
| 进度存储   | Redis                   | 高频更新，避免数据库压力              |

## 3. 架构设计

### 3.1 导入流程

```
用户上传 Excel 文件
        ↓
[文件校验] 格式、大小、表头校验
        ↓
创建 Task 记录，状态 = PENDING
        ↓
发送消息到 RabbitMQ import 队列
        ↓
立即返回 taskId 给用户（HTTP 202 Accepted）
        ↓
[异步消费] Spring Batch Job 启动
        ↓
┌───────────────────────────────────┐
│   EasyExcel 流式逐行读取           │
│   - 跳过已处理的分片               │
│   - 校验行数据格式                 │
│   - 累计满 1000 行批量写入数据库    │
│   - 每片更新 Redis 进度            │
└───────────────────────────────────┘
        ↓
更新 Task 状态 = SUCCESS / FAILED
        ↓
发送导入完成通知
```

### 3.2 分片策略

| 行数范围     | 分片大小 | 并发数 |
| ------------ | -------- | ------ |
| < 1 万行     | 1000 行  | 单线程 |
| 1万 - 10万行 | 5000 行  | 2 并发 |
| > 10 万行    | 10000 行 | 4 并发 |

## 4. 核心类设计

### 4.1 批处理分层架构

```
com.jinshu.batch
├── reader/              # 数据读取器
│   └── ExcelItemReader.java
├── processor/           # 数据处理器
│   └── ReportRowValidator.java
├── writer/              # 数据写入器
│   └── ReportBatchWriter.java
└── listener/            # 监听器
    ├── JobProgressListener.java
    └── JobFailureListener.java
```

### 4.2 任务状态流转

```
PENDING → PROCESSING → SUCCESS
                ↓
              FAILED (可重试)
```

| 状态       | 说明     |
| ---------- | -------- |
| PENDING    | 排队中   |
| PROCESSING | 导入中   |
| SUCCESS    | 导入完成 |
| FAILED     | 导入失败 |

## 5. 防 OOM 设计

### 5.1 关键措施

1. **EasyExcel 流式读取**：不将整个文件加载到内存
2. **批量写入后清空缓存**：每批次写入后手动 GC 提示
3. **JVM 参数配置**：`-Xmx512m -XX:+UseG1GC`
4. **并发控制**：同一租户最大同时导入任务数 = 2

### 5.2 内存估算

```
单行数据 ≈ 1KB
批量大小 = 1000 行 ≈ 1MB
4 并发 ≈ 4MB 内存占用
+ EasyExcel 缓冲区 ≈ 10MB
+ JVM 基础开销 ≈ 100MB
------------------------
峰值预估 ≈ 150MB < 512MB ✅
```

## 6. API 接口设计

### 6.1 上传导入

```http
POST /api/report/import
Content-Type: multipart/form-data

{
  "file": <excel-file>,
  "reportId": 1001
}

Response:
{
  "code": 0,
  "data": {
    "taskId": "task-12345"
  }
}
```

### 6.2 进度查询

```http
GET /api/task/{taskId}/progress

Response:
{
  "code": 0,
  "data": {
    "taskId": "task-12345",
    "status": "PROCESSING",
    "progress": 65,
    "processedRows": 65000,
    "totalRows": 100000,
    "failedRows": 0
  }
}
```

## 7. 异常处理

| 异常场景       | 处理方式                      |
| -------------- | ----------------------------- |
| 文件格式错误   | 同步返回，不创建任务          |
| 行数据校验失败 | 记录错误行号，继续导入后续行  |
| 数据库写入失败 | 重试 3 次，失败则标记任务失败 |
| 服务重启中断   | 基于分片记录，支持断点续传    |
